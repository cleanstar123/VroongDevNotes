## 주요 완료 작업

### 1. [Frontend] 협력사 → 라이더 프로모션 페이지 신규 구현

**배경**
- 협력사가 소속 라이더에게 직접 인센티브 미션을 부여하는 기능 필요
- 기간(하루/일주일) 동안 목표 건수 달성 시 보상 지급 방식

**미션 유형 3가지**

| 유형 | 코드 | 설명 |
|------|------|------|
| 초과분 단가 지급 | `OVERAGE` | 기간 동안 X건 이상 수행하면 초과 1건당 Y원 지급 |
| 목표 달성 고정 | `FIXED` | 기간 동안 X건 이상 수행하면 Y원 일시 지급 |
| 목표 달성 전체단가 | `FULL_UNIT` | 기간 동안 X건 이상 달성 시 전체 건수 × Y원 지급 |

**구현 내용**
- 미션 유형 카드 선택 UI (MISSION_TYPE 공통코드 연동, fallback 내장)
- 입력 폼: 제목·기간·목표건수·보상금액·최대인정건수·시작/종료일·설명
- 대상 라이더 선택: 전체 vs 특정 라이더만 (SPECIFIC 선택 시 라이더 목록 지연 로딩)
- 미션 미리보기 패널 (실시간 반영)
- 본사 계정 로그인 시 읽기전용 배너 표시 (등록 불가)
- 활성 미션 목록 테이블

**핵심 설계 포인트**
- 라이더 목록 API는 "특정 라이더만" 라디오 클릭 시 최초 1회만 호출 (마운트 시 호출 없음)
- 공통코드 API 실패해도 `FALLBACK_MISSION_TYPE_CODES`로 정상 동작

### 2. [Frontend] 활성 미션 상세 모달 + 종료 미션 섹션 추가

**활성 미션 행 클릭 → 상세 모달**
- 테이블 행 클릭 시 미션 상세 정보 모달 오픈
- 표시 항목: 미션 유형 배지, 제목, 상태, 기간·목표건수·보상·최대인정·시작/종료일·대상
- 대상 타입이 `SPECIFIC`이면 라이더 ID 칩 목록 표시
- 삭제 버튼은 `stopPropagation`으로 행 클릭 이벤트와 분리
- 모달 오픈 시 body 스크롤 잠금

**미션 제목 중복 방지**
- 등록 시 기존 미션 목록과 제목 비교 → 동일한 제목 존재 시 에러 토스트 표시 후 중단

**종료된 미션 섹션 신설**
- `endDate < today` 기준으로 종료 미션 필터링
- 카드형 리스트로 표시, 클릭 시 상세 모달 연결
- 분석 텍스트 포맷:
  ```
  A미션 (총 대상자 - X명 / 지급액 - Y원 || 일 대상자 - A명 / 일 평균 지급액 - B원)
  ```
  - 시작일 = 종료일인 **1일 미션**은 총 대상자/지급액만 노출 (`||` 이하 생략)

**활성 미션 필터 수정**
- 기존: `status === 'ACTIVE'`
- 변경: `status === 'ACTIVE' && endDate >= today` (종료일 지난 미션 활성 목록에서 제외)

### 3. [Bug Fix] 협력사 계정 로그인 시 suppId null 오류 수정

**증상**
- 협력사 계정으로 로그인 후 프로모션 페이지 진입 시 "협력사 정보를 찾을 수 없습니다." 에러 토스트 발생

**원인**
- 백엔드 로그인 API(`PartnerPortalLoginResponse`)가 응답 body에 `suppId` 필드를 포함하지 않음
- `vr_user_info` 테이블에 `SUPP_ID` 컬럼 없음 → localStorage에 저장되지 않음

**해결**
- 협력사 계정은 `userId === suppId` 관계가 DB(`vr_supplier_info.SUPP_ID = vr_user_info.USER_ID`)에서 보장됨
- `suppId` 없을 때 `userId`로 fallback

```typescript
// 변경 전
const suppId = userInfo?.suppId ?? null;

// 변경 후
const suppId = userInfo?.suppId ?? userInfo?.userId ?? null;
```

### 4. [Backend] 프로모션 미션 DB 테이블 설계 및 생성

**테이블 구조**

`vr_promo_mission` — 미션 마스터
```sql
SEQ_NO       BIGINT AUTO_INCREMENT  -- PK
SUPP_ID      VARCHAR(20)           -- 협력사 ID
TITLE        VARCHAR(200)          -- 미션 제목
MISSION_TYPE VARCHAR(20)           -- OVERAGE / FIXED / FULL_UNIT
PERIOD_TYPE  VARCHAR(10)           -- DAILY / WEEKLY
TARGET_CNT   INT                   -- 목표 건수
REWARD_AMT   BIGINT                -- 건당/달성 보상금액
MAX_CNT      INT                   -- 최대 인정 건수 (0=무제한)
START_DATE   DATE                  -- 미션 시작일
END_DATE     DATE                  -- 미션 종료일
DESCRIPTION  VARCHAR(500)          -- 설명
TARGET_TYPE  VARCHAR(10)           -- ALL / SPECIFIC
STATUS_CODE  VARCHAR(10)           -- ACTIVE / INACTIVE
INS_DATE, INS_USER, UPD_DATE, UPD_USER
```

`vr_promo_mission_rider` — 대상 라이더 매핑
```sql
SEQ_NO       BIGINT AUTO_INCREMENT  -- PK
MISSION_SEQ  BIGINT                -- vr_promo_mission.SEQ_NO FK
RIDER_ID     VARCHAR(50)           -- 라이더 ID
INS_DATE, INS_USER
```

### 5. [Backend] 프로모션 미션 API 구현

**엔드포인트** (`/api/v1/promotion/mission`)

| Method | Path | 설명 |
|--------|------|------|
| GET | `/list?suppId=` | 협력사의 미션 목록 조회 |
| POST | `/register` | 미션 신규 등록 |
| PUT | `/{seqNo}` | 미션 수정 |
| DELETE | `/{seqNo}` | 미션 삭제 (라이더 매핑 포함) |
| PATCH | `/{seqNo}/status` | 미션 상태 변경 (ACTIVE ↔ INACTIVE) |
