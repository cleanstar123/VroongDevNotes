### 2. 발견된 취약점

#### 🔴 고위험

**[B-01] CORS — 모든 Origin에 인증 자격증명 허용**

- **파일**: `api/src/main/java/com/vroong/api/app/security/SecurityConfig.java:66`
- **현황**:
  ```java
  configuration.setAllowedOriginPatterns(List.of("*"));  // 모든 도메인 허용
  configuration.setAllowedCredentials(true);              // 쿠키/인증 헤더 포함
  ```
- **위험**: `allowedOriginPatterns("*") + allowCredentials(true)` 조합은 악의적인 외부 사이트에서 쿠키/세션을 포함한 요청을 API 서버에 전송할 수 있습니다 (CSRF 유사 공격).
- **참고**: `ApiSecurityProperties.java`에 실제 허용 Origin 목록(`vrplatformacct.com`, `localhost`)이 이미 정의되어 있으나 `SecurityConfig.java`에서 사용되지 않고 있음.
- **조치**: `SecurityConfig.java`에서 와일드카드를 제거하고 허용 도메인을 명시:
  ```java
  configuration.setAllowedOriginPatterns(List.of(
      "http://localhost:3000",
      "http://localhost:8080",
      "https://www.vrplatformacct.com"
  ));
  ```

---

**[B-02] API Secret 기본값이 코드에 하드코딩**

- **파일**: `api/src/main/java/com/vroong/api/app/config/ApiSecurityProperties.java:14`
- **현황**:
  ```java
  private String secret = "default-secret-change-in-production";
  ```
- **위험**: `vroong.api.secret` 환경변수가 ECS 태스크 정의에 설정되어 있지 않으면 이 기본값으로 운영됩니다. 기본값은 공개된 코드에 노출되어 있어 API 인증이 무력화됩니다.
- **조치**: ECS 태스크 정의에 `VROONG_API_SECRET` 환경변수 설정 여부 즉시 확인 필요.

#### 🟡 중위험

**[B-03] SQL 문자열 직접 조합 (DDL)**

- **파일**: `api/src/main/java/com/vroong/api/infrastructure/persistence/repository/migration/TableBackupRepository.java:16,23,29`
- **현황**:
  ```java
  String sql = String.format("DROP TABLE IF EXISTS %s", tableName);
  String sql = String.format("CREATE TABLE %s AS SELECT * FROM %s", newTableName, sourceTableName);
  String sql = String.format("SELECT COUNT(*) FROM %s", tableName);
  ```
- **현재 위험도**: `TableBackupUseCase.java`에서 테이블명이 코드 상수로 하드코딩되어 있어 현재는 외부 입력이 없습니다. **즉각적인 SQL 인젝션 위험은 낮음.**
- **잠재 위험**: 향후 테이블명을 API 파라미터로 받도록 수정될 경우 즉시 SQL 인젝션 취약점이 됩니다. DDL은 파라미터 바인딩이 불가능하므로 화이트리스트 검증으로 방어:
  ```java
  private static final Set<String> ALLOWED_TABLES = Set.of(
      "vr_account_mst", "vr_rider_acct_daily", ...
  );

  private void validateTableName(String tableName) {
      if (!ALLOWED_TABLES.contains(tableName)) {
          throw new IllegalArgumentException("허용되지 않은 테이블명: " + tableName);
      }
  }
  ```

**[B-04] DB 연결 SSL 비활성화**

- **파일**: `application-*.yml`
- **현황**: 모든 환경의 JDBC URL에 `useSSL=false` 설정
- **현재 위험도**: AWS VPC 내부 통신이므로 외부 노출 가능성은 낮음. 단, 동일 VPC 내 다른 리소스가 침해된 경우 DB 트래픽 스니핑 가능.
- **조치**: 장기적으로 AWS RDS SSL 인증서를 적용하고 `useSSL=true`로 전환 권장.

---

**[B-05] Redis 인증 없음**

- **파일**: `api/src/main/resources/application-*.yml` (Redis 설정 섹션)
- **현황**: Redis 연결에 비밀번호 설정 없음 (VPC 네트워크 격리에만 의존)
- **저장 데이터**: 배민 로그인 세션(쿠키, 인증토큰) — 민감 데이터
- **조치**: AWS ElastiCache AUTH 토큰 설정 권장.

#### 🟢 낮음 / 참고

**[B-06] 자격증명 파일 Git 제외 — 정상**

- `application-local.yml`, `application-dev.yml`은 `.gitignore`에 등록되어 있어 원격 저장소에 노출되지 않음. ✅
- 단, 로컬 파일에 실제 prod RDS 엔드포인트/비밀번호가 기본값으로 포함되어 있어 개발자 PC 침해 시 노출 가능.

**[B-07] Swagger UI 인증 없이 공개**

- `/swagger-ui/**`, `/v3/api-docs/**` 경로가 인증 없이 접근 가능
- 외부 공개 서비스가 아닌 내부 운영 도구라면 수용 가능하나, 공개 인터넷에서 접근 가능한 경우 API 스펙 노출 위험.

### 4. 즉시 조치 가능 항목

**B-01 (CORS)** — 코드 3줄 수정, 배포 필요:

```java
// SecurityConfig.java:66 수정
configuration.setAllowedOriginPatterns(List.of(
    "http://localhost:3000",
    "http://localhost:8080",
    "https://www.vrplatformacct.com"
));
```

**B-02 (API Secret)** — ECS 환경변수 확인:

```bash
aws ecs describe-task-definition \
  --task-definition <current-task-def> \
  --query "taskDefinition.containerDefinitions[0].environment" \
  --region ap-northeast-2
```

`VROONG_API_SECRET`이 목록에 없다면 즉시 추가 필요.
