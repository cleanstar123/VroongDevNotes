# 주정산 일차감 프로세스 안내

> 주정산은 라이더 유형에 따라 계산 방식이 다릅니다.  
> - **선정산 라이더의 주정산**: 선정산에서 이미 일차감이 처리됨, 주정산은 정산 잔액 정리
> - **주정산 라이더의 주정산**: 주정산에서 일차감 전체를 한 번에 처리 (선정산 라이더와 동일한 4단계 프로세스)

---

# Part 1. 선정산 라이더의 주정산

## 개요

선정산 라이더는 **매일 배달료를 미리 지급**받으며, 이 과정에서 일차감이 매일 적용됩니다.  
주정산은 **일주일 치 배달료와 선정산 지급액을 최종 정리**하는 과정으로, 대부분 실지급액은 0원입니다.

주정산 일차감 금액(`DEDUCT_AMT`)은 **선정산에서 실제 적용된 금액 합계(실적값)** 에 **이번 주 선정산에서 회수된 이월 미수금(이전 주 미차감분)** 을 합산한 값입니다.

> 아래 모든 예시 기준: 일차감 단가 **7,000원 / 근무일**

---

## 기본 구조

```
주간 총 배달료
- 조정된 선정산 지급액 (= 선정산 지급액 - 이번 주 미수금 회수액)
= 주정산 정산 가능 금액
- 주정산 일차감 목표 (= 선정산 실적 일차감 + 이번 주 미수금 회수액)
= 주정산 실지급액 (대부분 0원)
- 이월 미수금 회수 (있는 경우)
= 실제 이체액
```

> **핵심**: 이번 주 선정산에서 이전 주 미수금을 회수한 경우, 그 회수액이 주정산 `DEDUCT_AMT`에 포함됩니다.  
> 이를 통해 협력사 정산 시 미수금 회수분이 일차감에 정확히 반영됩니다.

| 항목                      | 금액                   |
| ----------------------- | -------------------- |

→ 이번 주 선정산에서 회수된 7,000원이 `DEDUCT_AMT`에 포함되어 협력사 일차감으로 정확히 반영됩니다.

# Part 2. 주정산 라이더의 주정산

## 개요

주정산 라이더는 **매주 1회 정산** 시에만 배달료를 지급받습니다.  
일차감은 계약 규칙(`vr_deduct_mst`)을 기반으로 계산되어 주정산 배달료에서 차감됩니다.

선정산 라이더와 **동일한 4단계 프로세스**가 적용됩니다.

> 아래 모든 예시 기준: 일차감 단가 **10,000원 / 근무일** (5일 근무 시 50,000원)

---

## 기본 구조

```
실지급 가능 금액 = 주간 총 배달료 (주정산 라이더는 선정산 지급액 = 0)

① 이월 미수금 회수 (실지급 가능 금액에서 먼저 가능한 만큼 회수)
② 회수 후 남은 금액에서 이번 주 일차감 부분차감 (가능한 만큼 차감, 나머지는 신규 미수금)
= 실지급액

③ 신규 미수금이 발생한 경우:
   Vroong 계좌 잔액 ≥ 신규 미수금 전액 → 잔액에서 전액 차감 (부분 차감 없음)
   Vroong 계좌 잔액 < 신규 미수금 전액 → 다음 주로 이월

④ ③ 처리 후 잔액이 남아 있고, ①에서 미회수된 이월 미수금이 있는 경우:
   남은 잔액 범위 내에서 이월 미수금 잔여분 부분 회수 (부분 회수 허용)
```

> **이월 미수금 회수가 일차감보다 먼저 처리됩니다.**

---

## vr_rider_acct_weekly 컬럼 설명

| 컬럼 | 의미 | 계산식 |
|---|---|---|
| `DEDUCT_AMT` | 실제 차감된 일차감 총액 (배달료 + 잔액) | ② 배달료 차감 + ③ 잔액 차감 |
| `UNPAID_RECOVERY_AMT` | 배달료에서 차감된 일차감액 | ② `deductFromPay` |
| `BALANCE_DEDUCT_AMT` | 잔액에서 차감된 일차감액 | ③ `deductFromBalance` |
| `RECOVER_AMT` | 이전 미수금 총 회수액 (배달료 + 잔액) | ① `recoveredFromPay` + ④ `balRecoverAmt` |
| `BAL_RECOVER_AMT` | 잔액으로 회수한 이전 미수금 | ④ `balRecoverAmt` |

| **RECOVER_AMT**     | **20,000원**         |
| **실제 이체액**          | **430,000원**        |

→ 이월 미수금(20,000원)을 배달료에서 **먼저** 회수한 후, 잔여 배달료(480,000원)에서 일차감(50,000원)을 차감합니다.

| 일차감 차감 ②            | 0원 (잔여 배달료 없음)            |
| **실지급액**            | **0원**                     |
| 신규 미수금              | 50,000원                    |
| Vroong 계좌 잔액        | 0원                         |
| 잔액 차감 ③             | 0원                         |
| 잔액 추가회수 ④           | 0원                         |
| **DEDUCT_AMT**      | **0원**                     |
| **RECOVER_AMT**     | **300,000원**               |
| **미수금 재이월**         | **250,000원** (미회수 200,000 + 신규 50,000) |

→ 이월 미수금(500,000원) 중 300,000원 회수, 200,000원은 미회수.  
→ 이번 주 일차감은 배달료가 없어 전액 미수금(50,000원)으로 이월.

### Case B-6. 이월 미수금 — 배달료로 일부 회수 + 신규 미수금 누적

> 이월 미수금이 있는데 이번 주 배달료도 부족하여 미수금이 누적되는 경우 (잔액 없음)

| 일차감 차감 ②            | 0원 (잔여 배달료 없음)                 |

### Case B-7. 신규 미수금 — Vroong 계좌 잔액으로 전액 처리 (③)

> 이번 주 일차감이 부분 또는 전액 미차감되었지만 계좌 잔액이 충분한 경우

| 항목                  | 금액                           |
| ------------------- | ------------------------------ |
| 이월 미수금              | 0원                             |

| 미수금 회수 ①            | 0원                             |

### Case B-8. 이월 미수금 잔여분 — 잔액으로 부분 회수 (④)

> ①에서 이월 미수금 일부만 회수되고, ③ 처리 후 남은 잔액으로 잔여분을 추가 회수하는 경우

| 항목                  | 금액                                          |
| ------------------- | --------------------------------------------- |
| 이월 미수금              | 30,000원                                       |

→ 그러나 ③ 이후 남은 잔액(15,000원)으로 미회수 이월 미수금(10,000원)을 **부분 회수**합니다.  
→ 결과적으로 이전 이월 미수금 30,000원은 완전히 회수(20,000 + 10,000)되고, 신규 50,000원만 재이월됩니다.

## 전체 흐름 요약

```
주정산 실행
    │
    ├─ ① 이월 미수금 있음? (주간 배달료에서 먼저 회수)
    │       YES, 배달료 ≥ 미수금 → 전액 회수              (Case B-4)
    │       YES, 배달료 < 미수금 → 가능한 만큼만 회수, 나머지 잔여  (Case B-5, B-6, B-8)
    │       NO  → 생략
    │
    ├─ ② 이번 주 일차감 부분차감 (이월 미수금 회수 후 잔여 배달료 기준)
    │       잔여 배달료 ≥ 일차감 → 전액 차감               (Case B-1, B-4)
    │       잔여 배달료 < 일차감 → 가능한 만큼 차감, 나머지 신규 미수금  (Case B-2, B-3, B-7)
    │
    └─ 실지급 (항상 0원 이상)

    ③ 신규 미수금(이번 주 미차감분)이 있는 경우:
        계좌 잔액 ≥ 신규 미수금 전액 → 잔액에서 전액 차감      (Case B-7)
        계좌 잔액 < 신규 미수금 전액 → 재이월 (다음 주 미수금 누적) (Case B-2, B-3, B-8)

    ④ ①에서 배달료로 미회수된 이월 미수금 잔여분이 있고, ③ 처리 후 잔액이 남은 경우:
        남은 잔액 범위 내에서 이월 미수금 잔여분 부분 회수 (부분 회수 허용) (Case B-8)
```

## 핵심 원칙

1. **실지급액은 절대 음수가 되지 않습니다.**
2. **이월 미수금 회수가 이번 주 일차감보다 먼저 처리됩니다.**
3. **일차감은 부분차감 방식**: 잔여 배달료 범위 내에서 가능한 만큼 차감하고, 나머지는 미수금으로 이월합니다.
4. **잔액 기반 차감은 두 단계로 적용됩니다.**
   - **③ 신규 미수금**: 잔액이 신규 미수금 전액 이상일 때만 전액 차감합니다. (부분 차감 없음, full-or-nothing)
   - **④ 이월 미수금 잔여분**: ③ 처리 후 남은 잔액으로 배달료에서 미회수된 이월 미수금 잔여분을 추가 회수합니다. (부분 회수 허용)
5. **미수금은 해소될 때까지 매주 이월됩니다.**
6. **선정산 라이더의 이번 주 미수금 회수액은 주정산 DEDUCT_AMT에 포함되어 협력사 정산에 반영됩니다.**
