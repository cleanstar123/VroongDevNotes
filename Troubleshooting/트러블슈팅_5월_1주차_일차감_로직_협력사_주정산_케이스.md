# 협력사 주정산 일차감 프로세스 안내

---

## 개요

협력사 주정산은 Vroong이 협력사에 주간 단위로 정산을 처리하는 과정입니다.  
이때 소속 라이더들에게 적용된 일차감 내역이 협력사 정산액에 반영됩니다.

> **실행 순서**: 라이더 주정산 배치 → 협력사 주정산 배치  
> 협력사 주정산은 라이더 주정산이 완료된 이후 실행됩니다.

---

## 협력사 주정산 구조 (일반 협력사)

```
지급관리비(VAT 포함) + 일차감 - 보험료합계 = 정산액
```

| 항목      | 내용                                                     |
| ------- | ------------------------------------------------------ |
| 지급관리비   | (관리비 + 지점운영수수료 + 달성페이백) × 1.1 (VAT포함)                 |
| 일차감     | 해당 주간 소속 라이더 일차감 합계 (두 소스 합산, 아래 참고)                  |
| 보험료합계   | 사업주 고용보험 + 사업주 산재보험 (을지 시트 기준)                        |
| **정산액** | 지급관리비 + 일차감 - 보험료합계                                    |

---

## 일차감 집계 방식

협력사 주정산의 `일차감(deductVal)`은 **라이더별 주정산 레코드 유무**에 따라 두 소스에서 합산합니다.

```
deductVal = ① 주정산 레코드 있는 라이더의 vr_rider_acct_weekly.DEDUCT_AMT 합계
           + ② 주정산 레코드 없는 선정산 라이더의 vr_deduct_daily.DEDUCT_AMT 합계
```

### ① 주정산 레코드가 있는 라이더 (주정산 엑셀 포함)

라이더 주정산 배치(RiderSettlementUseCase) 실행 시 `vr_rider_acct_weekly`에 레코드가 생성된 라이더.

```sql
SELECT SUM(DEDUCT_AMT)
FROM vr_rider_acct_weekly
WHERE SUPPLY_ID = ?
  AND ACCT_ST_DATE >= ?
  AND ACCT_ED_DATE <= ?
  AND RIDER_ID IN (...)   -- 주정산 레코드가 존재하는 등록 라이더
```

| 라이더 유형          | DEDUCT_AMT 산정 방식                                                                              |
| --------------- | ------------------------------------------------------------------------------------------------- |
| 선정산 라이더 (엑셀 포함) | 선정산 실적 일차감 + 이번 주 선정산에서 회수된 이전 미수금 (배달료 차감 + 잔액 차감 모두 포함)                              |
| 주정산 라이더         | 배달료 차감액 + 잔액 차감액 (② + ③ 단계 합계). 미차감분은 미수금으로 이월되어 다음 주에 반영                               |

#### 주정산 라이더 vr_rider_acct_weekly 컬럼 구성

| 컬럼                  | 의미                                          |
| ------------------- | --------------------------------------------- |
| `DEDUCT_AMT`        | 실제 차감된 일차감 총액 = `UNPAID_RECOVERY_AMT` + `BALANCE_DEDUCT_AMT` |
| `UNPAID_RECOVERY_AMT` | 배달료에서 차감된 일차감액 (② 단계)                      |
| `BALANCE_DEDUCT_AMT`  | 잔액에서 차감된 일차감액 (③ 단계, full-or-nothing)     |
| `RECOVER_AMT`         | 이전 미수금 총 회수액 = 배달료 회수 (① 단계) + 잔액 회수 (④ 단계) |
| `BAL_RECOVER_AMT`     | 잔액으로 회수한 이전 미수금 (④ 단계)                     |

### ② 주정산 레코드가 없는 선정산 라이더 (주정산 엑셀 미포함)

선정산은 받지만 주정산 엑셀에 포함되지 않아 `vr_rider_acct_weekly` 레코드가 없는 라이더.  
이 경우 `vr_deduct_daily`에서 직접 실차감액을 합산합니다.

```sql
SELECT SUM(DEDUCT_AMT)
FROM vr_deduct_daily
WHERE SUPP_ID = ?
  AND TRAN_DAY BETWEEN ? AND ?
  AND RIDER_ID IN (...)   -- 주정산 레코드가 없는 등록 라이더
```

---

## 라이더별 소스 분류 흐름

```
등록 라이더 전체 (registeredRiderIds)
    │
    ├─ vr_rider_acct_weekly 레코드 있음?
    │       YES → vr_rider_acct_weekly.DEDUCT_AMT  ①
    │       NO  → vr_deduct_daily.DEDUCT_AMT        ②
    │
    └─ deductVal = ① + ②
```

---

## 공제 규칙 유형별 계산 (공제 마스터 기준)

공제 마스터(`vr_deduct_mst`)의 `DEDUCT_TARGET` 필드로 공제 대상이 구분됩니다.

### R 타입 — 특정 라이더 대상

| 계산 방식    | 공식                                 |
| -------- | ---------------------------------- |
| WORK_DAY | 공제금액 × 실제 근무일수 (공제기간 ∩ 정산기간 교집합 내) |
| DAILY    | 공제금액 × 교집합 일수 (공제 시작~종료일 ∩ 정산기간)   |

### SRA 타입 — 협력사 소속 전체 라이더 대상

| 계산 방식    | 공식                                               |
| -------- | ------------------------------------------------ |
| WORK_DAY | 공제금액 × 각 라이더별 근무일수의 합산                          |
| DAILY    | 공제금액 × 교집합 일수 × 배달 실적 있는 라이더 수                   |

### S 타입 — 협력사 자체 대상 (riderId 없음)

| 계산 방식 | 공식                       |
| ----- | ------------------------ |
| DAILY | 공제금액 × 정산기간 전체 일수       |

> S 타입은 `vr_deduct_daily`에 riderId=null로 기록되며, `vr_rider_acct_weekly`에는 기록되지 않습니다.  
> **재계산(Recalculate) 시에만** S 타입 합계가 deductVal에 반영됩니다.  
> `deductVal = (① + ②) - S 합계`

## 케이스별 예시

> 기준: 정산기간 1주(7일), 일차감 단가 7,000원/근무일(WORK_DAY), 라이더 등록 완료

---

### Case 1. 일반 협력사 — 정상 집계 (모두 주정산 엑셀 포함)

> 선정산 라이더 1명, 주정산 라이더 1명, 각각 5일 근무, 둘 다 주정산 엑셀 포함

| 항목                     | 내용                                           |
| ---------------------- | -------------------------------------------- |
| 선정산 라이더 DEDUCT_AMT (①) | 7,000 × 5일 = 35,000원 (vr_rider_acct_weekly)  |
| 주정산 라이더 DEDUCT_AMT (①) | 7,000 × 5일 = 35,000원 (vr_rider_acct_weekly, 배달료에서 전액 차감) |
| **일차감(합계)**            | **70,000원**                                  |
| 지급관리비                  | 1,210,000원                                   |
| 보험료합계                  | 50,000원                                      |
| **정산액**                | **1,210,000 + 70,000 - 50,000 = 1,230,000원** |

---

### Case 2. 일반 협력사 — 선정산 라이더가 주정산 엑셀 미포함

> 선정산 라이더 1명이 주정산 엑셀에 없어 vr_rider_acct_weekly 레코드 미생성

| 항목                                 | 내용                   |
| ---------------------------------- | -------------------- |
| 선정산 라이더 — vr_deduct_daily 합계 (②)   | 7,000 × 5일 = 35,000원 |
| 주정산 라이더 — vr_rider_acct_weekly (①) | 7,000 × 5일 = 35,000원 |
| **일차감(합계)**                        | **70,000원**          |

→ 선정산 라이더가 주정산 엑셀에 없더라도 `vr_deduct_daily.DEDUCT_AMT`에서 실차감액을 직접 합산하여 누락을 방지합니다.

---

### Case 3. 일반 협력사 — 선정산 라이더 배달 부족으로 부분차감

> 선정산 라이더(주정산 엑셀 포함)가 금요일에 배달료 부족으로 일차감 부분차감

| 항목                                 | 내용                                          |
| ---------------------------------- | ------------------------------------------- |
| 선정산 라이더 — 월~목 (4일)                | 정상 차감: 7,000 × 4일 = 28,000원                 |
| 선정산 라이더 — 금 (배달 3,000원)           | 부분차감: 3,000원 차감, 4,000원 미수금 이월              |
| 선정산 라이더 DEDUCT_AMT (①, vr_rider_acct_weekly) | **31,000원**                      |
| 주정산 라이더 DEDUCT_AMT (①)            | 35,000원                                     |
| **일차감(합계)**                       | **66,000원**                                 |

→ 미수금으로 이월된 4,000원은 다음 주 선정산에서 회수될 때 `vr_deduct_daily.DEDUCT_AMT`가 소급 업데이트됩니다.  
→ 회수가 이번 주에 이루어지면 `preRecoverAmt`를 통해 이번 주 `vr_rider_acct_weekly.DEDUCT_AMT`에 반영됩니다.

---

### Case 4. 주정산 라이더 — 잔액으로 일차감 처리

> 주정산 라이더의 배달료가 일차감에 부족하여 잔액에서 추가 차감된 경우

| 항목                     | 내용                               |
| ---------------------- | ---------------------------------- |
| 주정산 라이더 — 주간 배달료       | 56,068원                           |
| 주정산 일차감 목표             | 70,000원                           |
| 배달료 차감 ②               | 56,068원 (UNPAID_RECOVERY_AMT)     |
| 신규 미수금                 | 13,932원                           |
| 잔액 차감 ③ (잔액 ≥ 13,932원) | 13,932원 (BALANCE_DEDUCT_AMT)      |
| **DEDUCT_AMT (①)**    | **70,000원** (56,068 + 13,932)     |
| **일차감(합계)**            | **70,000원**                       |

→ `DEDUCT_AMT = UNPAID_RECOVERY_AMT + BALANCE_DEDUCT_AMT` = 배달료 차감 + 잔액 차감 = 실제 수취한 전체 일차감.  
→ 협력사 입장에서는 70,000원 전액이 정산에 반영됩니다.

---

### Case 5. 일반 협력사 — S 타입 공제 있는 경우 (재계산 시)

> 협력사 자체 공제(S 타입) 20,000원/주 설정, 재계산 실행 시

| 항목                 | 내용                                |
| ------------------ | --------------------------------- |
| 소속 라이더 DEDUCT_AMT 합계 (① + ②) | 70,000원                  |
| S 타입 공제            | 20,000원 (협력사 레벨, 7일 × 단가)         |
| **일차감**            | **70,000 - 20,000 = 50,000원**     |

→ 재계산(Recalculate) 기능 실행 시에만 `(① + ②) - S 합계` 방식이 적용됩니다.  
→ 일반 신규 배치 실행 시에는 S 타입이 deductVal에서 제외됩니다.

## 라이더 주정산과의 차이점

| 항목           | 라이더 주정산            | 협력사 주정산                              |
| ------------ | ------------------- | -------------------------------------- |
| 일차감 계산 대상    | 개별 라이더              | 협력사 소속 전체 라이더 합산                      |
| 유보(Deferral) | 있음 (배달료 부족 시 미수금 이월) | **없음** (라이더 레벨에서 처리된 값을 SUM)          |
| 계산 소스        | 공제 마스터 규칙 직접 계산     | vr_rider_acct_weekly + vr_deduct_daily |
| 미수금 개념       | 있음 (라이더 개인 미수금 추적)  | **없음** (협력사 레벨에서 추가 관리 없음)            |
| S 타입 공제      | 해당 없음               | 재계산 시 합산에서 차감                         |

---

## 핵심 원칙

1. **협력사 주정산은 라이더 주정산 완료 후 실행됩니다.**  
   라이더 주정산 배치가 `vr_rider_acct_weekly.DEDUCT_AMT`에 최종 일차감을 기록한 이후 합산합니다.

2. **주정산 레코드 유무로 소스를 분리합니다.**  
   - `vr_rider_acct_weekly` 레코드 있음 → `DEDUCT_AMT` 사용 (선정산/주정산 라이더 모두)  
   - `vr_rider_acct_weekly` 레코드 없음 → `vr_deduct_daily.DEDUCT_AMT` 직접 합산 (주정산 엑셀 미포함 선정산 라이더)

3. **선정산 라이더가 주정산 엑셀에 없어도 일차감이 누락되지 않습니다.**  
   `vr_deduct_daily`에서 해당 기간의 실차감액을 직접 합산하여 보완합니다.

4. **주정산 라이더의 `DEDUCT_AMT`는 배달료 차감 + 잔액 차감의 합계입니다.**  
   - `DEDUCT_AMT = UNPAID_RECOVERY_AMT(배달료 차감) + BALANCE_DEDUCT_AMT(잔액 차감)`  
   - 미수금으로 이월된 금액은 다음 주에 회수되면 그 주의 `DEDUCT_AMT`에 포함됩니다.

5. **선정산 라이더의 이번 주 미수금 회수액은 주정산 `DEDUCT_AMT`에 포함됩니다.**  
   이번 주 선정산에서 이전 주 미수금을 회수한 경우, 회수액이 `weeklyDeductTarget`에 합산되어 협력사 정산에 정확히 반영됩니다.

6. **S 타입 공제는 신규 배치에서 제외되며, 재계산(Recalculate) 기능 실행 시에만 deductVal에 반영됩니다.**
