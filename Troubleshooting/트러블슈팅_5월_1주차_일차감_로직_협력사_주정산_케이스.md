# 협력사 주정산 일차감 프로세스 안내

---

## 개요

협력사 주정산은 Vroong이 협력사에 주간 단위로 정산을 처리하는 과정입니다.  
이때 소속 라이더들에게 적용된 일차감 내역이 협력사 정산액에 반영됩니다.

> **실행 순서**: 라이더 주정산 배치 → 협력사 주정산 배치  
> 협력사 주정산은 라이더 주정산이 완료된 이후 실행되므로, `vr_rider_acct_weekly`에 저장된 최종값을 그대로 합산합니다.

---

## 협력사 주정산 구조 (일반 협력사)

```
지급관리비(VAT 포함) + 일차감 - 보험료합계 = 정산액
```

| 항목      | 내용                                        |
| ------- | ----------------------------------------- |
| 지급관리비   | (관리비 + 지점운영수수료 + 달성페이백) × 1.1 (VAT포함)     |
| 일차감     | 해당 주간 소속 라이더 일차감 합계 (vr_rider_acct_weekly.DEDUCT_AMT 기반) |
| 보험료합계   | 사업주 고용보험 + 사업주 산재보험 (을지 시트 기준)            |
| **정산액** | 지급관리비 + 일차감 - 보험료합계                       |

---

## 일차감 집계 방식

협력사 주정산의 `일차감(deductVal)`은 `vr_rider_acct_weekly` 테이블에서 해당 주간 소속 라이더들의 **DEDUCT_AMT 합계**를 조회합니다.

```sql
SELECT SUM(DEDUCT_AMT)
FROM vr_rider_acct_weekly
WHERE SUPPLY_ID = ?
  AND ACCT_ST_DATE >= ?
  AND ACCT_ED_DATE <= ?
  AND RIDER_ID IN (...)   -- vr_user_info 등록 라이더만
```

### 라이더 유형별 DEDUCT_AMT 저장 방식

라이더 주정산 배치가 실행되면서 모든 라이더의 일차감이 `vr_rider_acct_weekly.DEDUCT_AMT`에 최종 집계됩니다.

| 라이더 유형  | DEDUCT_AMT 산정 방식                                   |
| ------- | -------------------------------------------------- |
| 선정산 라이더 | `vr_deduct_daily.DEDUCT_AMT` 주간 합계 (실차감액, 부분차감 포함) |
| 주정산 라이더 | 공제 마스터 규칙 기반 이론 계산값 (배달 있는 날 근무일수 × 규칙금액)          |

> 협력사 주정산 deductVal = 선정산 라이더 DEDUCT_AMT + 주정산 라이더 DEDUCT_AMT  
> (라이더 주정산 배치가 이미 두 유형 모두를 `vr_rider_acct_weekly`에 기록)

---

## 공제 규칙 유형별 계산 (공제 마스터 기준)

공제 마스터(`vr_deduct_mst`)의 `DEDUCT_TARGET` 필드로 공제 대상이 구분됩니다.

### R 타입 — 특정 라이더 대상

| 계산 방식    | 공식                                 |
| -------- | ---------------------------------- |
| WORK_DAY | 공제금액 × 실제 근무일수 (공제기간 ∩ 정산기간 교집합 내) |
| DAILY    | 공제금액 × 교집합 일수 (공제 시작~종료일 ∩ 정산기간)   |

### SRA 타입 — 협력사 소속 전체 라이더 대상

| 계산 방식    | 공식                                               |
| -------- | ------------------------------------------------ |
| WORK_DAY | 공제금액 × 각 라이더별 근무일수의 합산                          |
| DAILY    | 공제금액 × 교집합 일수 × 배달 실적 있는 라이더 수                   |

### S 타입 — 협력사 자체 대상 (riderId 없음)

| 계산 방식 | 공식                       |
| ----- | ------------------------ |
| DAILY | 공제금액 × 정산기간 전체 일수       |

> S 타입은 `vr_deduct_daily`에 riderId=null로 기록되며, `vr_rider_acct_weekly`에는 기록되지 않습니다.  
> **재계산(Recalculate) 시에만** S 타입 합계가 deductVal에 반영됩니다.  
> `deductVal = SUM(DEDUCT_AMT) - S 합계`

## 케이스별 예시

> 기준: 정산기간 1주(7일), 일차감 단가 7,000원/근무일(WORK_DAY), 라이더 등록 완료

---

### Case 1. 일반 협력사 — 정상 집계

> 선정산 라이더 1명, 주정산 라이더 1명, 각각 5일 근무

| 항목                 | 내용                                           |
| ------------------ | -------------------------------------------- |
| 선정산 라이더 DEDUCT_AMT | 7,000 × 5일 = 35,000원 (vr_deduct_daily 합계)    |
| 주정산 라이더 DEDUCT_AMT | 7,000 × 5일 = 35,000원 (이론 계산)                 |
| **일차감(합계)**        | **70,000원**                                  |
| 지급관리비              | 1,210,000원                                   |
| 보험료합계              | 50,000원                                      |
| **정산액**            | **1,210,000 + 70,000 - 50,000 = 1,230,000원** |

---

### Case 2. 일반 협력사 — 선정산 라이더 배달 부족으로 일부 유보

> 선정산 라이더가 금요일에 배달료 부족으로 일차감 일부를 미수금으로 이월 (부분차감)

| 항목                         | 내용                                          |
| -------------------------- | ------------------------------------------- |
| 선정산 라이더 — 월~목 (4일)         | 정상 차감: 7,000 × 4일 = 28,000원                 |
| 선정산 라이더 — 금 (배달 3,000원)    | 부분차감: 3,000원 차감, 4,000원 미수금 이월              |
| 선정산 라이더 DEDUCT_AMT 합계      | **31,000원** (vr_deduct_daily.DEDUCT_AMT 합계) |
| 주정산 라이더 DEDUCT_AMT (5일 정상) | 35,000원 (이론 금액)                             |
| **일차감(합계)**                | **66,000원**                                 |

→ 금요일 부분차감분(3,000원)은 `vr_deduct_daily.DEDUCT_AMT`에 3,000원으로 기록되어 협력사 주정산 deductVal에 반영됩니다.  
→ 미수금으로 이월된 4,000원은 차후 회수 시 해당 일의 `vr_deduct_daily.DEDUCT_AMT`가 소급 업데이트됩니다.

---

### Case 3. 일반 협력사 — S 타입 공제 있는 경우 (재계산 시)

> 협력사 자체 공제(S 타입) 20,000원/주 설정, 재계산 실행 시

| 항목                   | 내용                                |
| -------------------- | --------------------------------- |
| 소속 라이더 DEDUCT_AMT 합계 | 70,000원 (vr_rider_acct_weekly 기준) |
| S 타입 공제              | 20,000원 (협력사 레벨, 7일 × 단가)         |
| **일차감**              | **70,000 - 20,000 = 50,000원**     |

→ 재계산(Recalculate) 기능 실행 시에만 `SUM(DEDUCT_AMT) - S 합계` 방식이 적용됩니다.  
→ 일반 신규 배치 실행 시에는 S 타입이 deductVal에서 제외됩니다.

---

## 라이더 주정산과의 차이점

| 항목         | 라이더 주정산            | 협력사 주정산                     |
| ---------- | ------------------- | ----------------------------- |
| 일차감 계산 대상  | 개별 라이더              | 협력사 소속 전체 라이더 합산             |
| 유보(Deferral) | 있음 (배달료 부족 시 미수금 이월) | **없음** (라이더 레벨에서 처리된 값을 SUM) |
| 계산 소스      | 공제 마스터 규칙 직접 계산     | vr_rider_acct_weekly.DEDUCT_AMT 합산 |
| 미수금 개념     | 있음 (라이더 개인 미수금 추적)  | **없음** (협력사 레벨에서 추가 관리 없음)   |
| S 타입 공제    | 해당 없음               | 재계산 시 DEDUCT_AMT 합산에서 차감      |

---

## 핵심 원칙

1. **협력사 주정산은 라이더 주정산 완료 후 실행됩니다.**  
   라이더 주정산 배치가 `vr_rider_acct_weekly.DEDUCT_AMT`에 모든 라이더(선정산/주정산)의 최종 일차감을 기록하면, 협력사 주정산은 이를 SUM합니다.

2. **선정산 라이더와 주정산 라이더를 구분하지 않습니다.**  
   `vr_rider_acct_weekly.DEDUCT_AMT`에 두 유형 모두의 값이 올바르게 집계되어 있으므로, 협력사는 소속 라이더 전체를 단순 합산합니다.

3. **선정산 라이더의 미수금 이월분은 당일 회수 시 vr_deduct_daily가 소급 업데이트됩니다.**  
   회수된 금액은 `vr_rider_acct_weekly.DEDUCT_AMT`에 포함되어 협력사 주정산에 반영됩니다.

4. **S 타입 공제는 신규 배치에서 제외되며, 재계산(Recalculate) 기능 실행 시에만 deductVal에 반영됩니다.**
