# 협력사 주정산 일차감 프로세스 안내

---

## 개요

협력사 주정산은 Vroong이 협력사에 주간 단위로 정산을 처리하는 과정입니다.  
이때 소속 라이더들에게 적용된 일차감 내역이 협력사 정산액에 반영됩니다.

협력사는 **일반 협력사**와 **가맹 협력사**로 구분되며, 일차감 처리 방식이 다릅니다.

---

## 협력사 유형별 주정산 구조

### 일반 협력사

```
지급관리비(VAT 포함) + 일차감 - 보험료합계 = 정산액
```

| 항목         | 내용                                    |
| ---------- | ------------------------------------- |
| 지급관리비     | (관리비 + 지점운영수수료 + 달성페이백) × 1.1 (VAT포함) |
| 일차감        | 해당 주간 소속 라이더 실차감액 합계 (vr_deduct_daily 기반) |
| 보험료합계     | 사업주 고용보험 + 사업주 산재보험 (을지 시트 기준)      |
| **정산액**   | 지급관리비 + 일차감 - 보험료합계                  |

---

### 가맹 협력사

```
공급대가 - 선정산금액 - 보험료합계 - 부릉수수료 = 정산액
```

| 항목       | 내용                           |
| -------- | ---------------------------- |
| 공급대가     | 공급가액(배달료 + 추가정산 + 관리비) + VAT |
| 선정산금액    | 해당 주간 선지급된 배달료 합계            |
| 보험료합계    | 사업주 고용보험 + 사업주 산재보험 + 기타보험   |
| 부릉수수료    | 처리건수 × 건당공제금액 × 1.1          |
| **일차감**  | **없음** (가맹 협력사는 일차감 미적용)     |
| **정산액**  | 공급대가 - 선정산금액 - 보험료합계 - 부릉수수료 |

---

## 일차감 집계 방식 (일반 협력사)

일반 협력사의 `일차감(deductVal)`은 `vr_deduct_daily` 테이블에서 해당 주간 소속 라이더들의 **실차감액 합계**를 조회합니다.

```sql
SELECT SUM(DEDUCT_AMT)
FROM vr_deduct_daily
WHERE SUPP_ID = ?
  AND TRAN_DAY BETWEEN ? AND ?
  AND RIDER_ID IS NOT NULL   -- 라이더 레벨 공제만 포함 (협력사 레벨 S타입 제외)
  AND RIDER_ID IN (...)      -- vr_user_info 등록 라이더만
```

### 라이더 유형별 vr_deduct_daily 기록 방식

| 라이더 유형  | 기록 시점     | 금액 산정 방식                            |
| -------- | --------- | ------------------------------------ |
| 선정산 라이더  | 매일 선정산 배치 | Greedy 알고리즘 결과 (규칙별 전부-or-전무 실차감액)  |
| 주정산 라이더  | 매일 이관 배치  | 이론적 금액 (배달 있는 날 규칙금액 × 해당일, 유보 없음) |

> 협력사 주정산 deductVal = 선정산 라이더 실차감액 합계 + 주정산 라이더 이론 차감액 합계

---

## 공제 규칙 유형별 계산 (공제 마스터 기준)

공제 마스터(`vr_deduct_mst`)의 `DEDUCT_TARGET` 필드로 공제 대상이 구분됩니다.

### R 타입 — 특정 라이더 대상

| 계산 방식    | 공식                                       |
| -------- | ---------------------------------------- |
| WORK_DAY | 공제금액 × 실제 근무일수 (공제기간 ∩ 정산기간 교집합 내)     |
| DAILY    | 공제금액 × 교집합 일수 (공제 시작~종료일 ∩ 정산기간)       |

### SRA 타입 — 협력사 소속 전체 라이더 대상

| 계산 방식    | 공식                                               |
| -------- | ------------------------------------------------ |
| WORK_DAY | 공제금액 × 각 라이더별 근무일수의 합산                          |
| DAILY    | 공제금액 × 교집합 일수 × 배달 실적 있는 라이더 수                   |

### S 타입 — 협력사 자체 대상 (riderId 없음)

| 계산 방식 | 공식                       |
| ----- | ------------------------ |
| DAILY | 공제금액 × 정산기간 전체 일수       |

> S 타입은 `vr_deduct_daily`에 riderId=null로 기록되며, 주정산 **신규 계산 시에는 제외**됩니다.  
> **재계산(Recalculate) 시에만** R/SRA 합계에서 S 타입 합계가 차감됩니다.  
> `deductVal = R/SRA 합계 - S 합계`

## 케이스별 예시

> 기준: 정산기간 1주(7일), 일차감 단가 7,000원/근무일(WORK_DAY), 라이더 등록 완료

---

### Case 1. 일반 협력사 — 정상 집계

> 선정산 라이더 1명, 주정산 라이더 1명, 각각 5일 근무

| 항목             | 내용                                    |
| -------------- | ------------------------------------- |
| 선정산 라이더 실차감액   | 7,000 × 5일 = 35,000원 (greedy 적용)     |
| 주정산 라이더 이론 차감액 | 7,000 × 5일 = 35,000원 (이론적 금액)        |
| **일차감(합계)**    | **70,000원**                           |
| 지급관리비          | 1,210,000원                            |
| 보험료합계          | 50,000원                               |
| **정산액**        | **1,210,000 + 70,000 - 50,000 = 1,230,000원** |

---

### Case 2. 일반 협력사 — 선정산 라이더 배달 부족으로 일부 유보

> 선정산 라이더가 배달료 부족으로 일차감 일부 유보 (잔액 차감도 불가)

| 항목                   | 내용                             |
| -------------------- | ------------------------------ |
| 선정산 라이더 — 월~목 (4일)   | 정상 차감: 7,000 × 4일 = 28,000원   |
| 선정산 라이더 — 금 (배달 3,000원) | 배달 부족 → 7,000원 전액 유보 (0원 차감) |
| 선정산 라이더 실차감액 합계       | **28,000원** (유보된 7,000원은 미포함) |
| 주정산 라이더 (5일 정상)       | 35,000원 (이론 금액)               |
| **일차감(합계)**           | **63,000원**                    |

→ 선정산 라이더가 금요일에 일차감을 유보했으므로, 해당 7,000원은 `vr_deduct_daily`에 0원(또는 미기록)으로 남아 협력사 주정산 deductVal에 포함되지 않습니다.

---

### Case 3. 일반 협력사 — S 타입 공제 있는 경우 (재계산 시)

> 협력사 자체 공제(S 타입) 20,000원/주 설정, 재계산 실행 시

| 항목        | 내용                      |
| --------- | ----------------------- |
| R/SRA 공제  | 70,000원                 |
| S 타입 공제   | 20,000원 (협력사 레벨, 7일 × 단가) |
| **일차감**   | **70,000 - 20,000 = 50,000원** |

→ 재계산(Recalculate) 기능 실행 시에만 `R/SRA 합계 - S 합계` 방식이 적용됩니다.  
→ 일반 신규 배치 실행 시에는 S 타입이 deductVal에서 제외됩니다.

---

### Case 4. 가맹 협력사 — 일차감 없음

> 가맹 협력사는 일차감(deductVal) = 0원, 별도 수식으로 정산

| 항목        | 내용           |
| --------- | ------------ |
| 공급대가      | 5,000,000원   |
| 선정산금액     | 4,000,000원   |
| 보험료합계     | 100,000원     |
| 부릉수수료     | 55,000원      |
| **일차감**   | **0원 (해당 없음)** |
| **정산액**   | **5,000,000 - 4,000,000 - 100,000 - 55,000 = 845,000원** |

---

## 라이더 주정산과의 차이점

| 항목         | 라이더 주정산            | 협력사 주정산                     |
| ---------- | ------------------- | ----------------------------- |
| 일차감 계산 대상  | 개별 라이더              | 협력사 소속 전체 라이더 합산             |
| 유보(Deferral) | 있음 (배달료 부족 시 이월)   | **없음** (라이더 레벨에서 처리된 값을 SUM) |
| 계산 소스      | 공제 마스터 규칙 직접 계산     | vr_deduct_daily 실적 합산        |
| 미수금 개념     | 있음 (라이더 개인 미수금 추적)  | **없음** (협력사 레벨에서 추가 관리 없음)   |
| S 타입 공제    | 해당 없음               | 재계산 시 R/SRA에서 차감             |
| 가맹/일반 구분   | 없음                  | 있음 (계산식 다름)                  |

---

## 핵심 원칙

1. **협력사 주정산 일차감은 라이더 레벨에서 이미 처리된 값을 단순 합산합니다.**  
   선정산/주정산 배치에서 각 라이더에 적용된 차감 결과를 주간 단위로 SUM하여 협력사 정산에 반영합니다.

2. **선정산 라이더가 일차감을 유보한 날은 협력사 deductVal에도 미반영됩니다.**  
   greedy 알고리즘 결과가 0원인 경우 vr_deduct_daily에 0원으로 기록되어 합산에서 영향이 없습니다.

3. **가맹 협력사는 일차감 프로세스가 없습니다.**  
   부릉수수료(건당공제)로 별도 관리됩니다.

4. **S 타입 공제는 신규 배치에서 제외되며, 재계산(Recalculate) 기능 실행 시에만 deductVal에 반영됩니다.**
