### 해결 방법

#### 1. Puppeteer 모드 활성화

**파일**: `relay-server/.env`

```env
# 변경 전
USE_PUPPETEER=false

# 변경 후
USE_PUPPETEER=true
```

#### 2. 쿠키 배열을 객체로 변환 함수 추가

**파일**: `relay-server/server.js`

```javascript
/**
 * Puppeteer 쿠키 배열을 객체 형식으로 변환
 * [{name: 'foo', value: 'bar'}, ...] => {foo: 'bar', ...}
 */
function convertCookiesToObject(cookies) {
  if (!cookies) return {};
  if (!Array.isArray(cookies)) return cookies;

  const result = {};
  for (const cookie of cookies) {
    if (cookie.name && cookie.value !== undefined) {
      result[cookie.name] = cookie.value;
    }
  }
  return result;
}
```

#### 3. relay-server 응답에 쿠키 포함

**파일**: `relay-server/server.js`

## 📝 교훈 및 개선 사항

### 근본 원인 추정

**배민 측 보안 강화로 추정**

- 예전에는 로컬에서 기존 코드로 로그인 → 2FA → 인증 완료까지 정상 동작
- 현재는 Cloudflare 봇 탐지가 강화되어 직접 API 호출 차단
- Puppeteer(실제 브라우저 자동화)로만 우회 가능

### 주요 교훈

1. **쿠키 형식 호환성**
   - Puppeteer와 Axios는 쿠키 형식이 다름
   - 시스템 간 데이터 교환 시 형식 변환 필요

2. **Redis 키 충돌**
   - 여러 시스템이 같은 Redis 키를 사용하면 덮어쓰기 위험
   - 각 시스템의 역할을 명확히 분리하거나 병합 로직 필요

3. **Cloudflare 우회**
   - 프록시 서비스도 제한이 있을 수 있음 (KYC 등)
   - Puppeteer가 더 안정적이나 리소스 소모 큼

- **브랜치명**: `feature/relay-login-cookies`
- **커밋 해시**: `403308d`
