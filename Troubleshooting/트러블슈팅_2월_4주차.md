## 3. 버그 수정

### 3-1. 어드민 계정 로그인 시 잔액 조회 N+1 문제

**현상**

어드민 계정 로그인 시 `vr_account_mst` 테이블 조회 쿼리가 라이더 수만큼 반복 실행되는 N+1 문제가 발견되었습니다. 로그 분석 결과, 라이더 목록 조회 시 각 라이더마다 개별적으로 잔액을 조회하는 패턴이 확인되었습니다.

**원인**

라이더 목록 렌더링 시 각 항목마다 별도의 잔액 조회 쿼리가 발생하는 구조였으며, 어드민(`prm_code = 'A'`) 계정에서도 불필요하게 잔액 컴포넌트가 렌더링되고 있었습니다.

**해결 방법**

**프론트엔드 (앱)**

- `DeliveryPage.tsx`에서 `prmCode='A'`인 경우 `BalanceDisplay` 컴포넌트 렌더링 제외
- 어드민 계정에서 불필요한 잔액 조회 쿼리 완전 제거

**백엔드 (포털 API)**

- `VrAccountMstQueryDslRepository`에 배치 조회 메서드 추가 (`findLatestBalAmtByRiderIds`)
- `RiderListController`에서 개별 조회 대신 배치 조회 방식으로 전환
- N+1 쿼리를 단일 IN 쿼리로 최적화

### 최종 결론

**AWS 인프라 레벨에서 자동 복구(Recovery)가 실행됨**

- `Connect timed out` = TCP 연결 자체가 안 됨 = MySQL 프로세스가 없거나 네트워크 불가
- RDS 이벤트에 "Recovery started" 기록 = AWS가 복구 프로세스 실행
- 사용자가 트리거한 작업 없음 (CloudTrail 기록 없음)

**복구가 필요했던 가능한 원인:**

|가능성|설명|확인 방법|
|---|---|---|
|하드웨어 장애|호스트 서버 문제로 자동 이전|AWS Support 문의|
|OS 패치/업데이트|AWS 내부 패치 적용|AWS Support 문의|
|메모리 부족 (OOM)|1GB RAM으로 MySQL 8.0 운영 중 OOM 발생 가능|CloudWatch 60초 간격으로 순간 스파이크 놓칠 수 있음|

정확한 원인 확인을 위해서는 AWS Support 유료 플랜(Technical Support) 필요.
