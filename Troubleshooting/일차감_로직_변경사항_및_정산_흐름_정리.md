## 1. 핵심 변경 사항 요약

### 문제였던 것

1. **음수 정산 발생**: 배달료보다 일차감이 크면 `TOT_ACCT_VAL`이 음수가 됐음
2. **미수금 미회수**: `vr_account_mst.UNPAID_AMT`에 저장은 되지만 다음 정산에서 읽지 않았음
3. **협력사 이론값 차감**: 라이더가 실제로 차감을 못 받았어도 협력사에는 이론값이 그대로 반영됐음
4. **선정산 라이더 차감 기록 없음**: `vr_deduct_daily`에 선정산 라이더 내역이 아예 없었음

### 해결된 것

| # | 해결 내용 | 관련 파일 |
|---|---|---|
| 1 | **Largest Fits First** 알고리즘으로 규칙별 전부-or-전무 적용 → 음수 완전 방지 | `BaeminToVroongIfUseCase.java` |
| 2 | 정산 저장 시 이전 미수금 자동 회수 | `BaeminToVroongIfUseCase.java`, `RiderSettlementUseCase.java` |
| 3 | 협력사 주정산 차감액을 `vr_deduct_daily.DEDUCT_AMT` 실적용액 합산으로 변경 | `PartnerSettlementUseCase.java` |
| 4 | 선정산 처리 시 `vr_deduct_daily`에 규칙별 실적용액/미납금 삽입 | `BaeminToVroongIfUseCase.java` |
| 5 | 주정산 처리 시 `vr_deduct_daily`에 기간 집계 1건 삽입 | `RiderSettlementUseCase.java` |

|---|---|---|

## 3. Largest Fits First 알고리즘

### 개념

- 모든 일차감 규칙을 **금액 내림차순** 정렬
- 각 규칙은 **전부 적용하거나 전부 보류** (부분 적용 없음)
- `remaining` (정산 가능 잔여 금액) 범위 내에서 큰 것부터 맞는 규칙 적용

### 알고리즘 흐름

```
baseAcctVal = 배달료 - 수수료 - 산재보험 - 고용보험 - 원천징수 - 시간제보험
remaining = max(baseAcctVal, 0)

일차감 규칙 목록 (정액 + 정률) 를 금액 내림차순 정렬

for 규칙 in 정렬된 목록:
    if remaining >= 규칙.금액:
        → DEDUCT_AMT = 규칙.금액,  UNPAID_AMT = 0        (적용)
        → remaining -= 규칙.금액
    else:
        → DEDUCT_AMT = 0,           UNPAID_AMT = 규칙.금액 (보류)

totAcctVal = baseAcctVal - sum(DEDUCT_AMT)   -- 항상 >= 0
```

## 4. 선정산 라이더 예시

### 기본 정보

- **라이더**: 박철수 (선정산 라이더)
- **협력사**: SUPP001
- **일차감 규칙**:
  - 규칙A (장비대여비): **30,000원** (정액)
  - 규칙B (교육비): **10,000원** (정액)
- **수수료+보험+세금 합계**: 배달료의 약 20% 가정 (단순화)

---

### 월요일 — 배달 많은 날

| 항목 | 금액 |
|---|---|
| 배달료 | 80,000원 |
| 수수료+보험+세금 | 15,000원 |
| **baseAcctVal** | **65,000원** |

**Largest Fits First 적용** (30,000 → 10,000 순):

| 규칙 | 금액 | remaining | 결과 |
|---|---|---|---|
| 규칙A (장비대여비) | 30,000 | 65,000 → 35,000 | **적용** |
| 규칙B (교육비) | 10,000 | 35,000 → 25,000 | **적용** |

**결과**:
- `DEDUCT_AMT 합계` = 40,000원
- `UNPAID_AMT 합계` = 0원
- `TOT_ACCT_VAL` = 65,000 - 40,000 = **25,000원** 지급

**vr_deduct_daily 삽입**:
```
TRAN_DAY=20260428, RIDER_ID=박철수, 장비대여비: DEDUCT_AMT=30000, UNPAID_AMT=0
TRAN_DAY=20260428, RIDER_ID=박철수, 교육비:    DEDUCT_AMT=10000, UNPAID_AMT=0
```

**vr_account_mst (RP)**:
```
TRAN_AMT=25,000  BAL_AMT=25,000  UNPAID_AMT=0
```

### 화요일 — 배달 적은 날

| 항목 | 금액 |
|---|---|
| 배달료 | 22,000원 |
| 수수료+보험+세금 | 7,000원 |
| **baseAcctVal** | **15,000원** |
| 이전 UNPAID_AMT | 0원 |

**Largest Fits First 적용** (30,000 → 10,000 순):

| 규칙 | 금액 | remaining | 결과 |
|---|---|---|---|
| 규칙A (장비대여비) | 30,000 | 15,000 | 15,000 < 30,000 → **보류** |

**결과**:
- `DEDUCT_AMT 합계` = 10,000원
- `UNPAID_AMT 합계` = 30,000원 (규칙A 전액 이월)
- `TOT_ACCT_VAL` = 15,000 - 10,000 = **5,000원** 지급

**vr_deduct_daily 삽입**:


### 수요일 — 배달 많은 날 + 미수금 회수

| 항목              | 금액           |
| --------------- | ------------ |
| 배달료             | 90,000원      |
| 수수료+보험+세금       | 15,000원      |
| **baseAcctVal** | **75,000원**  |
| 이전 UNPAID_AMT   | **-30,000원** |

**Largest Fits First 적용**:

| 규칙 | 금액 | remaining | 결과 |
|---|---|---|---|
| 규칙A (장비대여비) | 30,000 | 75,000 → 45,000 | **적용** |
| 규칙B (교육비) | 10,000 | 45,000 → 35,000 | **적용** |

- `DEDUCT_AMT 합계` = 40,000원, `UNPAID_AMT 합계` = 0원
- 일차감 후 잔액 = 75,000 - 40,000 = 35,000원

**미수금 회수 처리**:
```
unpaidAbs    = abs(-30,000) = 30,000원
recoveryAmt  = min(35,000, 30,000) = 30,000원   ← 미수금 전액 회수
실지급액     = 35,000 - 30,000 = 5,000원
newUnpaidAmt = -30,000 + 30,000 - 0 = 0원       ← 미수금 완전 해소
```

**vr_deduct_daily 삽입**:
```
TRAN_DAY=20260430, RIDER_ID=박철수, 장비대여비: DEDUCT_AMT=30000, UNPAID_AMT=0
TRAN_DAY=20260430, RIDER_ID=박철수, 교육비:    DEDUCT_AMT=10000, UNPAID_AMT=0
```

**vr_account_mst (RP)**:
```
TRAN_AMT=5,000   BAL_AMT=35,000  UNPAID_AMT=0   ← 미수금 해소
```

## 7. 전체 흐름 요약

```
[매일 선정산 배치] — BaeminToVroongIfUseCase
  ① 배달료/수수료/보험/세금 계산
  ② 일차감 규칙 조회 (vr_deduct_mst)
  ③ Largest Fits First 알고리즘 적용
      → List<DeductApplicationResult> (규칙별 actual/deferred)
  ④ vr_rider_acct_daily 저장
      DEDUCT_VAL = sum(actual)
      TOT_ACCT_VAL = baseAcctVal - sum(actual) ← 항상 >= 0
  ⑤ vr_deduct_daily 저장 (선정산 라이더, 규칙별 1행)
      기존 동일 날짜 삭제 후 재삽입 (멱등성)
  ⑥ 이전 UNPAID_AMT 조회 → 회수 처리
  ⑦ vr_account_mst (ACCT_TYPE=RP) 저장
      TRAN_AMT = TOT_ACCT_VAL - recoveryAmt
      UNPAID_AMT = 이전UNPAID + 회수액 - todayDeferred

[주 1회 주정산 엑셀 업로드] — RiderSettlementUseCase
  ① 엑셀에서 주간 배달료/선정산 금액 파싱
  ② 주간 일차감 합계 계산 (calculateDeductAmountOptimized)
  ③ 총합 cap 방식으로 actual/deferred 결정
      baseVal = finalDeliveryVal - preAcctVal
      actual  = min(baseVal, totalDeductAmt) ← 음수 방지
      deferred = totalDeductAmt - actual
  ④ vr_deduct_daily 저장 (기간 집계 1건, tranDay=정산종료일)
      DEDUCT_AMT=actual, UNPAID_AMT=deferred
  ⑤ 이전 UNPAID_AMT 조회 → 회수 처리
  ⑥ vr_account_mst (ACCT_TYPE=RW) 저장

[협력사 주정산] — PartnerSettlementUseCase
  ① vr_deduct_daily에서 기간 내 DEDUCT_AMT 합산 조회
      (선정산 라이더 일별 실적용액 + 주정산 라이더 집계 1건)
  ② DEDUCT_VAL = 실적용액 합계 (이론값 아님)
  ③ vr_supp_acct_weekly 저장
  ④ vr_account_mst (ACCT_TYPE=SW) 저장
```
