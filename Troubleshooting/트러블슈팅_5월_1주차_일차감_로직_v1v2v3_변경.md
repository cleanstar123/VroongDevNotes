# 일차감 정산 프로세스 안내

---

## 일차감이란?

일차감은 라이더가 협력사와 계약 시 설정된 금액을 **매주 정산 시 배달료에서 차감**하는 항목입니다.  
예시) 보험료, 장비 대여료, 유니폼 비용 등

차감 방식은 계약에 따라 두 가지로 구분됩니다.

| 유형 | 설명 | 예시 |
|---|---|---|
| **근무일 기준** (WORK_DAY) | 해당 주에 실제 배달을 수행한 날 수 × 단가 | 근무 4일 × 3,000원 = 12,000원 |
| **일수 기준** (DAILY) | 정산 기간의 달력 일수 × 단가 | 7일 × 1,500원 = 10,500원 |

---

## 기본 정산 구조

주정산 시 실지급 배달료는 아래 순서로 계산됩니다.

```
① 주간 총 배달료
② - 선정산액 (주 중에 미리 지급된 금액)
③ = 정산 가능 금액
④ - 일차감
⑤ = 실지급액
```

---

## 케이스별 예시

---

### Case 1. 정상 차감 (가장 일반적인 경우)

> 정산 가능 금액이 일차감보다 충분히 많은 경우

| 항목 | 금액 |
|---|---|
| 주간 총 배달료 | 500,000원 |
| 선정산액 | 200,000원 |
| 정산 가능 금액 | **300,000원** |
| 일차감 | 50,000원 |
| **실지급액** | **250,000원** |
| 미수금 발생 | 없음 |

→ 차감 전액이 적용되고 라이더는 250,000원을 지급받습니다.

---

### Case 2. 부분 차감 + 일부 유보 (미수금 발생)

> 정산 가능 금액이 일차감보다 적지만 0보다는 큰 경우

| 항목 | 금액 |
|---|---|
| 주간 총 배달료 | 500,000원 |
| 선정산액 | 470,000원 |
| 정산 가능 금액 | **30,000원** |
| 일차감 | 50,000원 |
| 실제 적용된 차감 | 30,000원 (가능한 금액만) |
| **실지급액** | **0원** |
| **미수금 (다음 주로 이월)** | **20,000원** |

→ 정산 가능 금액인 30,000원까지만 차감하고, 부족한 **20,000원은 미수금으로 이월**됩니다.  
→ 실지급액은 0원 미만으로 내려가지 않습니다.

---

### Case 3. 전액 유보 (차감 불가)

> 선정산액이 배달료 전액에 해당하여 정산 가능 금액이 0인 경우

| 항목 | 금액 |
|---|---|
| 주간 총 배달료 | 500,000원 |
| 선정산액 | 500,000원 |
| 정산 가능 금액 | **0원** |
| 일차감 | 50,000원 |
| 실제 적용된 차감 | 0원 |
| **실지급액** | **0원** |
| **미수금 (다음 주로 이월)** | **50,000원** |

→ 이번 주는 차감이 전혀 이루어지지 않고, **50,000원 전액이 다음 주로 이월**됩니다.

---

### Case 4. 다음 주 — 이월 미수금 배달료에서 회수

> Case 2 또는 3에서 미수금이 발생한 다음 주 정산

| 항목 | 금액 |
|---|---|
| (이전 주 이월 미수금) | 20,000원 |
| 주간 총 배달료 | 500,000원 |
| 선정산액 | 200,000원 |
| 정산 가능 금액 | **300,000원** |
| 이번 주 일차감 | 50,000원 |
| 이월 미수금 회수 | 20,000원 |
| **실지급액** | **230,000원** |

→ 이번 주 정산 가능 금액에서 **이번 주 일차감(50,000원) + 이월 미수금(20,000원)** 을 함께 차감합니다.  
→ 정산 가능 금액이 충분할 경우 미수금은 당주에 전액 회수됩니다.

---

### Case 5. 잔액 기반 미수금 회수 (Full-or-Nothing)

> 배달료로 미수금 회수가 불가할 때, Vroong 예치 잔액에서 회수하는 경우

이번 주도 정산 가능 금액이 부족하여 이월 미수금을 배달료에서 회수할 수 없는 상황입니다.  
이 경우 라이더의 **Vroong 계좌 잔액**을 활용하여 회수를 시도합니다.

**조건: 잔액이 미수금 전액 이상일 때만 차감 (부분 차감 없음)**

#### 잔액이 충분한 경우 → 잔액에서 전액 차감

| 항목 | 금액 |
|---|---|
| 이월 미수금 | 50,000원 |
| 주간 총 배달료 | 500,000원 |
| 선정산액 | 480,000원 |
| 정산 가능 금액 | **20,000원** (미수금 회수 불가) |
| 이번 주 일차감 | 50,000원 → 전액 유보 |
| Vroong 계좌 잔액 | **60,000원** (≥ 이월 미수금 50,000원) |
| 잔액 차감 | 50,000원 |
| **실지급액** | **20,000원** |
| 미수금 잔여 | 0원 (완전 회수) |

→ 잔액(60,000원)이 이월 미수금(50,000원) 이상이므로 잔액에서 전액 차감하여 미수금을 해소합니다.

#### 잔액이 부족한 경우 → 이번 주는 회수 안 함, 다음 주로 재이월

| 항목 | 금액 |
|---|---|
| 이월 미수금 | 50,000원 |
| Vroong 계좌 잔액 | **30,000원** (< 이월 미수금 50,000원) |
| 잔액 차감 | **0원** (부분 차감 없음) |
| **다음 주로 재이월** | **50,000원** |

→ 잔액이 부족할 경우 **이번 주는 회수하지 않고 전액을 다음 주로 재이월**합니다.  
→ 일부만 차감하는 방식은 사용하지 않습니다. (라이더 혼란 방지 및 계산 일관성)

---

## 선정산 라이더의 일차감 흐름

선정산 라이더는 매일 배달 완료 시점에 배달료를 미리 지급받습니다.  
이 과정에서 **일차감이 매일 실시간으로 적용**되며, 주정산 시에는 선정산 과정에서 실제로 적용된 금액을 그대로 사용합니다.

```
월요일 선정산: 배달료 80,000원 → 일차감 7,000원 적용 → 지급 73,000원
화요일 선정산: 배달료 90,000원 → 일차감 7,000원 적용 → 지급 83,000원
...
금요일 선정산: 배달료 60,000원 → 일차감 전액 유보 (배달료 부족)
```

→ 주정산 시: 이미 선정산에서 실제로 차감된 금액 합산값을 기준으로 정산  
→ 이론상 차감액이 아닌 **실제 적용된 금액**을 사용하므로 중복 차감이 발생하지 않습니다.

---

## 전체 흐름 요약

```
주정산 실행
    │
    ├─ 정산 가능 금액 = 배달료 - 선정산액
    │
    ├─ 정산 가능 금액 ≥ 일차감?
    │       YES → 전액 차감, 미수금 없음
    │       NO  → 가능한 금액만 차감, 나머지 미수금 이월
    │
    ├─ 이전 주 이월 미수금 있음?
    │       YES → 이번 주 정산 가능 금액에서 우선 회수
    │             불가 시 → Vroong 잔액 확인
    │                       잔액 ≥ 미수금 전액 → 잔액에서 전액 차감
    │                       잔액 < 미수금 전액 → 다음 주로 재이월
    │
    └─ 실지급 (항상 0원 이상)
```

---

## 핵심 원칙

1. **실지급액은 절대 음수가 되지 않습니다.**
2. **미수금은 다음 정산 주기로 이월되어 반드시 회수됩니다.**
3. **잔액 기반 차감은 전액 회수 가능할 때만 실행됩니다.** (부분 차감 없음)
4. **선정산 라이더는 실제 차감된 금액 기준으로 주정산이 이루어집니다.**
