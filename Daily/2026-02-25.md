## 1. 라이더 상세 정보 개별 트랜잭션 저장

> [!info] 배경
> - 라이더 상세 정보 배치 작업 중 일부 라이더 조회가 403 에러로 실패
> - 로그에는 "저장 완료"가 출력되지만 실제 DB에 데이터가 없는 현상 발생

> [!warning] 문제
> - `RiderDetailInfoUseCase.getRiderDetailInfo()` 메서드가 `@Transactional`로 전체가 하나의 트랜잭션
> - for 루프에서 각 라이더 저장 시 로그는 출력되지만, 실제 커밋은 메서드 종료 시점
> - 중간에 하나라도 에러(403 등) 발생 시 **전체 트랜잭션 롤백** → 이미 저장한 데이터도 모두 삭제

> [!success] 해결
> - `RiderDetailInfoSaveService` 클래스 신규 생성
> - `@Transactional(propagation = Propagation.REQUIRES_NEW)` 적용하여 각 라이더 저장이 독립 트랜잭션으로 처리
> - `RiderDetailInfoUseCase`에서 개별 라이더 조회/저장을 try-catch로 감싸서 실패해도 계속 진행
> - 성공/실패 건수 로깅 추가

> [!tip] 배운 점
> - `@Transactional` 메서드 내에서 `save()` 호출 후 로그 출력해도 실제 DB 커밋은 메서드 종료 시점
> - 개별 트랜잭션이 필요한 경우 별도 서비스 클래스에서 `REQUIRES_NEW` propagation 사용
> - 같은 클래스 내 메서드 호출은 프록시를 거치지 않아 트랜잭션 적용 안됨

### 변경 파일
- `api/src/main/java/com/vroong/api/domain/rider/usecase/RiderDetailInfoSaveService.java` (신규)
- `api/src/main/java/com/vroong/api/domain/rider/usecase/RiderDetailInfoUseCase.java` (수정)
