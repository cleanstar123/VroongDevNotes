## 1. vr_account_mst UNPAID_AMT (미수금) 전면 처리

> [!info] 배경
> - `vr_account_mst`에 입금/출금 기록이 생성될 때 이전 기록의 `UNPAID_AMT`(미수금)가 무시되고 있었음
> - 미수금이 있는 상태에서 입금이 들어와도 잔액 계산에 미수금이 반영되지 않아 BAL_AMT가 부정확했음

> [!warning] 문제
> - **D타입 입금**: `BAL = prevBAL + depositAmt` 로 계산되어 이전 미수금 무시
> - **W/F타입 출금**: 새 레코드에 `UNPAID_AMT` 컬럼이 NULL로 저장되어 미수금 이력 단절
> - **TO/TI 송금**: TO는 발신자 미수금 미이월, TI는 D타입과 동일한 미수금 무시 문제

> [!success] 해결
> **공통 유효잔액 공식 (D타입, TI타입 적용)**
> ```
> effectiveBal = prevBAL + prevUNPAID + depositAmt
> newBAL  = effectiveBal >= 0 ? effectiveBal : 0
> newUNPAID = effectiveBal < 0  ? effectiveBal : null
> ```
>
> **W/F타입, TO타입 (출금/송금 발신)**
> - 잔액 계산 변경 없음
> - 이전 기록의 `UNPAID_AMT`를 그대로 새 레코드에 이월
> ```java
> .unpaidAmt(prevUnpaid != 0 ? prevUnpaid : null)
> ```
>
> **수정 파일**
> - `HectoDepositUseCase.java` — D타입 헥토 입금 미수금 고려 잔액 계산
> - `MobWithdrlUseCase.java` — D타입 모바일 입금, W/F 출금, TO/TI 송금 전체 처리
>
> **RECOVER_AMT 반영 (`vr_rider_acct_daily`)**
> - D/TI 입금으로 미수금이 해소될 때 `VrRiderAcctDailyQueryDslRepository.addRecoverAmtForDeposit()` 호출
> - 완전 해소: `amountRecovered = |prevEffectiveBal|`
> - 부분 해소: `amountRecovered = depositAmt`
> - 당일 일별 기록 없으면 가장 최근 일별 기록에 반영 (선정산 전 케이스 대응)

> [!tip] 배운 점
> - `prevUNPAID`는 음수 저장 (e.g. 빚 30,000원 → `-30,000`), `vr_deduct_daily.UNPAID_AMT`는 양수 저장으로 부호 주의
> - RP 배치의 `tranAmt`(TOT_ACCT_VAL)는 이미 미수금을 고려한 실수령액이므로 RP 잔액 계산은 수정 불필요

## 2. vr_deduct_daily 미수금 회수 소급 반영

> [!info] 배경
> - D/TI 입금으로 미수금이 해소되면 `vr_account_mst.UNPAID_AMT`는 0이 되지만
>   원인이 된 `vr_deduct_daily` 레코드의 `DEDUCT_AMT=0, UNPAID_AMT=30000` 상태가 그대로 남아 있었음

> [!warning] 문제
> ```
> [기존] vr_deduct_daily
> SEQ  TRAN_DAY  DEDUCT_AMT  UNPAID_AMT
> 6702  20260517        0      30,000   ← 입금 후에도 미회수 상태 유지
> ```

> [!success] 해결
> - 이미 구현된 `VrDeductDailyQueryDslRepository.updateDeductDailyForRecovery()` 를 D/TI 입금 처리 시 호출
> - FIFO 방식: 오래된 UNPAID 레코드부터 순서대로 `DEDUCT_AMT += covered`, `UNPAID_AMT -= covered`
>
> **회수금 계산**
> - 완전 해소: `deductRecovery = prevUnpaidAmt.negate()` (= |UNPAID|)
> - 부분 해소: `deductRecovery = depositAmt`
>
> ```
> [수정 후] vr_deduct_daily
> SEQ  TRAN_DAY  DEDUCT_AMT  UNPAID_AMT
> 6702  20260517    30,000        0     ← 정상 반영
> ```
>
> **수정 파일**
> - `HectoDepositUseCase.java` — `VrDeductDailyQueryDslRepository` 의존성 추가 및 D타입 입금 후 호출
> - `MobWithdrlUseCase.java` — D타입 입금, TI타입 송금 입금 후 호출

> [!tip] 배운 점
> - RECOVER_AMT(`vr_rider_acct_daily`)와 deduct_daily 회수는 기준 금액이 다름
>   - RECOVER_AMT: `|prevEffectiveBal|` (prevBAL + prevUNPAID 의 절대값)
>   - deduct_daily: `|prevUNPAID|` (미수금 원금 그대로)
