## 3. 자동 재인증 시스템 구축

> [!info] 배경
> - 배민 로그인 시 2FA(SMS 인증) 필요
> - MacroDroid 앱으로 SMS 수신 → webhook 전달 → 자동 인증 완료

> [!warning] 문제
> - SMS 웹훅이 relay-server에 도달하지 않음
> - `eq(null) is not allowed. Use isNull() instead` 에러 발생

> [!success] 해결
>
> **1. MacroDroid URL 설정**
> - 잘못된 URL: `https://www.vrplatformacct.com/webhook/sms`
> - 올바른 URL: `http://vroong-alb-xxx.ap-northeast-2.elb.amazonaws.com:8091/webhook/sms?message={sms_body}`
> - www.vrplatformacct.com은 CloudFront 경유로 8091 포트 접근 불가
>
> **2. vroong-api-service에 RELAY_SERVER_URL 환경변수 추가 필요 없음**
> - ALB를 통해 직접 relay-server에 접근하는 방식으로 해결

> [!tip] 배운 점
> - ECS에서 같은 클러스터 내 다른 서비스는 localhost로 통신 불가
> - 같은 Task 내 컨테이너만 localhost 통신 가능
> - 서비스 간 통신은 Service Discovery, ALB, 또는 Private IP 필요

## 4. ALB 8091 포트 리스너 및 ECS Service 연동

> [!info] 배경
> - 기존 8090 포트는 dev relay-server용 (`vroong-relay-tg-dev`)
> - prod relay-server용 별도 포트 필요

> [!warning] 문제
> - 8091 리스너 생성 후 보안 그룹에서 포트 차단
> - 타겟 그룹에 타겟 미등록
> - Health check 실패 (Request timed out)
> - ECS Task 재배포 시 Private IP 변경되어 수동 업데이트 필요

> [!success] 해결
>
> **1. ALB 설정**
> - 8091 리스너 추가
> - 보안 그룹에 8091 포트 인바운드 규칙 추가 (0.0.0.0/0)
>
> **2. 타겟 그룹 설정**
> - `vroong-relay-tg` 생성
> - VPC: `vpc-02e3505b445d19786`
> - Health check path: `/health`
> - Port: 8090
>
> **3. ECS Service에 Load Balancer 연동**
> - vroong-relay-service → Update → Load balancing 설정
> - Load balancer: 기존 ALB
> - Listener: 8091:HTTP
> - Target group: vroong-relay-tg
> - 이제 Task 재배포 시 자동으로 타겟 등록/해제

> [!tip] 배운 점
> - ECS Service에 ALB를 직접 연동하면 Task IP 변경을 자동 관리
> - 타겟 그룹 Health check 실패 시 확인 순서:
>   1. 타겟 등록 여부
>   2. 보안 그룹 인바운드 규칙
>   3. Health check path/port 설정
>   4. 가용 영역 포함 여부
>   5. VPC 설정
