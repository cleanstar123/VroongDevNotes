## 1. 메일 서비스 오류 수정 - send401Alert 함수 누락

> [!info] 배경
> - prod 브랜치에서 메일 서비스가 동작하지 않는 문제 발생
> - `baemin/utils.js`의 `handleApiError`에서 401 에러 시 `mailService.send401Alert()` 호출

> [!warning] 문제
> - `mail-service.js`에 `send401Alert` 함수가 정의되지 않음
> - `TypeError: mailService.send401Alert is not a function` 런타임 에러 발생
> - 전체 에러 처리 로직 실패로 모든 메일 발송 중단

> [!success] 해결
> - `mail-service.js`에 `send401Alert` 함수 추가
> - `getSubjectByErrorType`, `getDescriptionByErrorType`에 401 케이스 추가
> - `module.exports`에 `send401Alert` export 추가

## 3. 비크리티컬 에러 메일 발송 정책 변경

> [!info] 배경
> - 502 에러 등 비크리티컬 에러로 인한 메일이 너무 자주 발송됨
> - 배달현황 조회, 라이더 상세 정보 조회에서 특정 센터 조회 실패 메일 과다

> [!warning] 문제
> - 일시적인 네트워크 오류에도 즉시 메일 발송
> - 메일 피로감 증가, 중요한 알림(세션 만료) 놓칠 수 있음

> [!success] 해결
> - 에러 타입 분류:
>   - **크리티컬**: SESSION_EXPIRED, SESSION_INACTIVE, 403 → 즉시 알림
>   - **비크리티컬**: 401, 500, 502, 503, 504 → 3회 연속 실패 시 알림
> - `consecutiveFailureMap` 추가하여 센터+API별 연속 실패 카운트 추적
> - `shouldSendNonCriticalAlert()` 함수로 발송 여부 결정
> - 3회 도달 시 메일 발송 후 카운트 리셋

```javascript
// 동작 예시
502 에러 (센터 A, 배달현황 조회) - 1회째: 로그만
502 에러 (센터 A, 배달현황 조회) - 2회째: 로그만
502 에러 (센터 A, 배달현황 조회) - 3회째: 메일 발송!
```

## 4. 개발 서버 환경변수 문제 수정

> [!warning] 문제 1: MAIL_HOST 잘못된 설정
> - 설정값: `MAIL_HOST = cleanstar1212@umac.co.kr` (이메일 주소)
> - 에러: `queryA EBADNAME cleanstar1212@umac.co.kr` (DNS 조회 실패)
> - 원인: SMTP 서버 주소가 아닌 이메일 주소 설정

> [!success] 해결
> - `MAIL_HOST = smtp.umac.co.kr` (또는 회사 SMTP 서버 주소)

> [!warning] 문제 2: NODE_ENV 설정
> - 개발 서버인데 `NODE_ENV = production`으로 설정됨
> - 메일에 "운영서버"로 표시됨

> [!success] 해결
> - AWS ECS 개발 서비스에서 `NODE_ENV = development`로 변경
