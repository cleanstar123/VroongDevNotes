### riderDeliveryHistoriesNowJob / riderDeliveryHistoriesNowIfJob 주기 변경

기존 13개 트리거(평일/주말 구분) → 새로운 7개 트리거로 단순화

| 시간대 | 간격 | Cron 표현식 |
|--------|------|-------------|
| 00:00~02:59 | 10분 | `0 0/10 0-2 * * ?` |
| 03:00~03:30 | 10분 | `0 0,10,20,30 3 * * ?` |
| 03:31~05:59 | 조회 안함 | - |
| 06:00~11:59 | 10분 | `0 0/10 6-11 * * ?` |
| 12:00~13:59 | 3분 | `0 0/3 12-13 * * ?` |
| 14:00~16:59 | 10분 | `0 0/10 14-16 * * ?` |
| 17:00~19:59 | 3분 | `0 0/3 17-19 * * ?` |
| 20:00~23:59 | 10분 | `0 0/10 20-23 * * ?` |

> [!tip] 배운 점
> - Quartz 스케줄러는 DB(QRTZ_TRIGGERS, QRTZ_CRON_TRIGGERS)에 트리거 정보를 저장
> - 코드 변경 시 DB도 함께 변경해야 함 (UPDATE 또는 DELETE 후 재시작)
> - QRTZ_TRIGGERS → QRTZ_CRON_TRIGGERS 순서로 외래키 관계가 있음

## 2. 출금제한 로직 변경 (라이더/협력사)

> [!info] 변경 배경
> - 기존: 배치에서 마이너스 잔액인 협력사/라이더에 출금제한 자동 설정
> - 문제점: 배치 실행 전까지 출금이 가능해 타이밍 이슈 발생
> - 해결: 앱에서 이미 마이너스 잔액 시 출금 버튼 비활성화 → 배치 자동 제한 불필요

### 변경된 정책

| 대상 | 변경 전 | 변경 후 |
|------|---------|---------|
| **협력사** | 마이너스 잔액 시 배치에서 출금제한 설정 | 상시 출금 허용 (앱에서 잔액 검증) |
| **라이더** | 마이너스 잔액 시 배치에서 출금제한 설정 | 상시 출금 허용, **입금 당일만 제한** |

> [!important] 라이더 입금 당일 제한 이유
> - 가상계좌 입금 후 즉시 출금 시 정산 오류 가능성
> - 입금 당일만 `WITHDRAWAL_YN = 'Y'` 설정
> - 다음날 00:05 배치에서 자동 해제

### 수정된 파일

| 파일 | 위치 | 변경 내용 |
|------|------|----------|
| `BatchWithdrawalUseCase.java` | vroong-backend | 자동 출금제한 로직 제거, 라이더 해제만 유지 |
| `HectoDepositUseCase.java` | vroong-backend | 라이더 입금 시 `WITHDRAWAL_YN='Y'` 설정 추가 |
| `BatchWithdrawalQuartzConfig.java` | vroong-3pl-platform-batch | 스케줄 변경: 매시 정각 → 매일 00:05 |

### 주요 코드 변경

#### 1. BatchWithdrawalUseCase.java

```java
/**
 * 라이더 출금제한 해제 처리 (입금 당일 제한 해제)
 *
 * 변경된 출금 제한 정책:
 * - 라이더: 상시 출금 허용, 단 가상계좌 입금 당일만 제한 (입금 다음날 이 배치에서 해제)
 * - 협력사: 상시 출금 허용 (자동 제한 없음)
 * - 앱에서 마이너스 잔액 시 출금 버튼이 비활성화되므로 별도 출금제한 불필요
 */
@Transactional
public BatchWithdrawalResult processBatchWithdrawal() {
    // 협력사/라이더 자동 출금제한 처리 제거
    log.info("협력사/라이더 자동 출금제한 처리 건너뜀 (상시 출금 허용 정책)");

    // 라이더 출금제한 해제 처리 (입금 당일 제한 해제)
    List<BatchWithdrawalTarget> riderTargetsToRelease =
        vrAccountMstQueryDslRepository.findRidersWithPositiveBalanceAndWithdrawalYn();

    for (BatchWithdrawalTarget target : riderTargetsToRelease) {
        vrRiderInfoQueryDslRepository.releaseWithdrawalYn(
            target.getSupplyId(), target.getRiderId(), updDate);
    }
}
```

#### 시나리오 3: 라이더 입금 당일 앱에서 출금 실패

> [!tip] 개발 서버에서 실제 입금 불가능한 경우
> - `WITHDRAWAL_YN = 'Y'`를 **수동으로 설정**하여 테스트 가능
> - 실제 입금 동작 없이도 출금 제한 동작 검증 가능

```sql
-- 라이더 출금제한 수동 설정 (입금 당일 상황 시뮬레이션)
UPDATE vr_rider_info
SET WITHDRAWAL_YN = 'Y',
    UPD_DATE = NOW()
WHERE SUPP_ID = '{테스트 협력사 ID}' AND RIDER_ID = '{테스트 라이더 ID}';

-- 라이더 계좌에 양수 잔액 설정 (출금 시도 가능하도록)
UPDATE vr_account_mst
SET BAL_AMT = 100000
WHERE SUPP_ID = '{테스트 협력사 ID}'
  AND RIDER_ID = '{테스트 라이더 ID}'
  AND ACCT_TARGET = 'R';
```

**앱 테스트**:
1. 해당 라이더로 앱 로그인
2. 출금 페이지 접근
3. 출금 시도

**예상 결과**: "출금이 제한된 계정입니다" 메시지 표시 ✅
