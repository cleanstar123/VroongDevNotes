## 1. 배치 실행 주기 변경 (prod 브랜치)

> [!info] 배경
> - 테스트용으로 30분/10분 간격으로 설정되어 있던 배치들을 운영 환경에 맞게 변경 필요
> - 피크 시간대(12~14시, 17~20시)에는 더 자주 실행되도록 조정

> [!warning] 변경 대상
> - PartnerQuartzConfig.java
> - RiderInfoQuartzConfig.java
> - DeliveryQuartzConfig.java
> - MigrationQuartzConfig.java

> [!success] 변경 내역

### 단순 주기 변경

| Job | 변경 전 | 변경 후 |
|-----|---------|---------|
| partnerDeliveryFeeJob | 10시, 13시 | 9시, 10시 (하루 2회) |
| partnerDetailJob | 30분마다 | 매일 14시 (하루 1회) |
| riderInfoJob | 30분마다 | 매시 정각 |
| deliveryDailyHistoriesJob | 30분마다 | 9시, 10시 (하루 2회) |
| riderDeliveryHistoriesJob | 30분마다 | 9시, 10시 (하루 2회) |
| riderInfoIfJob | 10분마다 | 매시 5분 |
| partnerDetailsIfJob | 10분마다 | 매시 5분 |
| partnerDeliveryFeesIfJob | 10분마다 | 매시 5분 |
| riderDeliveryHistoriesIfJob | 10분마다 | 매시 5분 |

### riderDeliveryHistoriesNowJob / riderDeliveryHistoriesNowIfJob 주기 변경

기존 13개 트리거(평일/주말 구분) → 새로운 7개 트리거로 단순화

| 시간대 | 간격 | Cron 표현식 |
|--------|------|-------------|
| 00:00~02:59 | 10분 | `0 0/10 0-2 * * ?` |
| 03:00~03:30 | 10분 | `0 0,10,20,30 3 * * ?` |
| 03:31~05:59 | 조회 안함 | - |
| 06:00~11:59 | 10분 | `0 0/10 6-11 * * ?` |
| 12:00~13:59 | 3분 | `0 0/3 12-13 * * ?` |
| 14:00~16:59 | 10분 | `0 0/10 14-16 * * ?` |
| 17:00~19:59 | 3분 | `0 0/3 17-19 * * ?` |
| 20:00~23:59 | 10분 | `0 0/10 20-23 * * ?` |

> [!tip] 배운 점
> - Quartz 스케줄러는 DB(QRTZ_TRIGGERS, QRTZ_CRON_TRIGGERS)에 트리거 정보를 저장
> - 코드 변경 시 DB도 함께 변경해야 함 (UPDATE 또는 DELETE 후 재시작)
> - QRTZ_TRIGGERS → QRTZ_CRON_TRIGGERS 순서로 외래키 관계가 있음

### DB 변경 쿼리

#### 1. 단순 UPDATE (Cron 표현식 변경)

```sql
-- partnerDeliveryFeeTrigger: 10시,13시 → 9시,10시
UPDATE QRTZ_CRON_TRIGGERS
SET CRON_EXPRESSION = '0 0 9,10 * * ?'
WHERE TRIGGER_NAME = 'partnerDeliveryFeeTrigger' AND TRIGGER_GROUP = 'partnerGroup';

-- partnerDetailTrigger: 30분마다 → 매일 14시
UPDATE QRTZ_CRON_TRIGGERS
SET CRON_EXPRESSION = '0 0 14 * * ?'
WHERE TRIGGER_NAME = 'partnerDetailTrigger' AND TRIGGER_GROUP = 'partnerGroup';

-- riderInfoTrigger: 30분마다 → 매시 정각

WHERE TRIGGER_NAME = 'riderInfoTrigger' AND TRIGGER_GROUP = 'riderGroup';

-- deliveryDailyHistoriesTrigger: 30분마다 → 9시,10시
UPDATE QRTZ_CRON_TRIGGERS
SET CRON_EXPRESSION = '0 0 9,10 * * ?'
WHERE TRIGGER_NAME = 'deliveryDailyHistoriesTrigger' AND TRIGGER_GROUP = 'deliveryGroup';

-- riderDeliveryHistoriesTrigger: 30분마다 → 9시,10시

SET CRON_EXPRESSION = '0 0 9,10 * * ?'

-- riderInfoIfTrigger: 10분마다 → 매시 5분
UPDATE QRTZ_CRON_TRIGGERS
SET CRON_EXPRESSION = '0 5 * * * ?'

-- partnerDetailsIfTrigger: 10분마다 → 매시 5분

SET CRON_EXPRESSION = '0 5 * * * ?'

-- riderDeliveryHistoriesIfTrigger: 10분마다 → 매시 5분
UPDATE QRTZ_CRON_TRIGGERS
SET CRON_EXPRESSION = '0 5 * * * ?'
WHERE TRIGGER_NAME = 'riderDeliveryHistoriesIfTrigger' AND TRIGGER_GROUP = 'migrationGroup';
```
