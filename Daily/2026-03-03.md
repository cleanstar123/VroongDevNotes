## 1. 출금제한 로직 분석

> [!info] 배경
> - 협력사가 계약종료(TERMINATED) 상태일 때 소속 라이더들도 출금제한에 걸리는지 문의
> - 표준경북경산A부릉라이더스2 (DP2509124244) 케이스 분석

> [!success] 분석 결과
> - **협력사 상태(TERMINATED)와 출금제한은 직접적인 관계 없음**
> - 출금제한은 **잔액(BAL_AMT)**을 기준으로 처리
> - 협력사: 음수 잔액 시 출금제한 설정, **자동 해제 불가** (수동만 가능)
> - 라이더: 협력사 출금제한 시 함께 제한, 양수 잔액 + 협력사 출금허용 시 자동 해제

**출금제한 배치 주기**: 매시 정각 (1시간마다)

### 출금제한 흐름

```
1. 협력사 잔액 음수 → 협력사 WITHDRAWAL_YN = 'Y'
2. 해당 협력사 소속 라이더도 모두 WITHDRAWAL_YN = 'Y'
3. 라이더 잔액 양수 + 협력사 출금허용(N) → 라이더 자동 해제
4. 협력사는 자동 해제 안됨 → 수동 해제 필요
```

## 2. relay-server Redis 재연결 후 세션 복구 로직 추가

> [!info] 배경
> - 2026-03-03 11:35:26 Redis 연결 오류 발생 (SocketClosedUnexpectedlyError)
> - Redis 재연결 후 `active_sessions` Set이 비어있어 세션 0개 상태
> - 재로그인(자동 재인증)이 트리거되지 않아 배치 실패

> [!warning] 문제
> - `keepAllSessionsAlive()` 함수에서 세션이 0개일 때 그냥 return
> - browserManager에 세션이 있어도 active_sessions에 없으면 감지 못함
> - 자동 재인증이 트리거되지 않음

```javascript
// 기존 코드 (문제)
if (activeSessionIds.length === 0) return;  // 그냥 종료!
```

> [!success] 해결
> - 세션 0개일 때 browserManager에서 세션 확인
> - browserManager에 세션 있으면 → active_sessions에 복구
> - browserManager에도 없으면 → 자동 재인증 시작

```javascript
// 수정된 코드
if (activeSessionIds.length === 0) {
    if (USE_PUPPETEER) {
        const localSessions = browserManager.getAllSessions ? browserManager.getAllSessions() : [];
        const authenticatedSessions = localSessions.filter(s => s.loginStatus === 'AUTHENTICATED');

        if (authenticatedSessions.length > 0) {
            // browserManager에 세션이 있으면 active_sessions에 복구
            console.log(`[Keep-Alive] Redis 세션 0개, browserManager에 ${authenticatedSessions.length}개 발견 - 복구 시도`);
            for (const session of authenticatedSessions) {
                await redisClient.sAdd(`${REDIS_KEY_PREFIX}active_sessions`, session.sessionId);
                console.log(`[Keep-Alive] 세션 복구: ${session.sessionId.substring(0, 8)}...`);
            }
            return; // 복구 후 다음 주기에서 정상 처리
        } else {
            // browserManager에도 세션이 없으면 자동 재인증 시작
            console.log('[Keep-Alive] Redis 및 browserManager 모두 세션 없음 - 자동 재인증 시작');
            if (autoReauthService.isEnabled() && !autoReauthService.getStatus().isInProgress) {
                autoReauthService.startReauth()
                    .then(result => {
                        if (result.success) {
                            console.log(`[Keep-Alive] 자동 재인증 성공: ${result.sessionId}`);
                        } else {
                            console.error(`[Keep-Alive] 자동 재인증 실패: ${result.error}`);
                        }
                    })
                    .catch(err => {
                        console.error('[Keep-Alive] 자동 재인증 오류:', err.message);
                    });
            }
        }
    }
    return;
}
```

> [!tip] 배운 점
> - Redis 연결 끊김 시 메모리(browserManager)와 Redis(active_sessions) 상태가 불일치할 수 있음
> - 세션 복구 로직이 여러 단계로 필요 (Redis 확인 → 메모리 확인 → 재인증)

### 수정 후 동작

| 상황 | 동작 |
|------|------|
| Redis 세션 0개 + browserManager에 세션 있음 | active_sessions에 복구 |
| Redis 세션 0개 + browserManager에도 없음 | 자동 재인증 시작 |

### 수정 파일

```
relay-server/server.js - keepAllSessionsAlive() 함수
```

### 출금제한 관련 SQL

```sql
-- 협력사 출금제한 상태 확인
SELECT SUPPLY_ID, WITHDRAWAL_YN FROM vr_supplier_info WHERE SUPPLY_ID = 'DP2509124244';

-- 협력사 잔액 확인
SELECT SUPPLY_ID, BAL_AMT FROM vr_account_mst
WHERE SUPPLY_ID = 'DP2509124244' AND ACCT_TARGET = 'S'
ORDER BY SEQ_NO DESC LIMIT 1;

-- 라이더 중 잔액 양수인데 출금제한인 경우 조회
SELECT a.RIDER_ID, a.SUPPLY_ID, a.BAL_AMT
FROM vr_account_mst a
INNER JOIN (
    SELECT RIDER_ID, MAX(SEQ_NO) AS max_seq
    FROM vr_account_mst WHERE ACCT_TARGET = 'R'
    GROUP BY RIDER_ID
) latest ON a.RIDER_ID = latest.RIDER_ID AND a.SEQ_NO = latest.max_seq
INNER JOIN vr_rider_info r ON a.RIDER_ID = r.RIDER_ID
WHERE a.BAL_AMT > 0 AND r.WITHDRAWAL_YN = 'Y';
```

### Redis 세션 관련

- `active_sessions` Set: 활성 세션 ID 목록
- `session:{sessionId}`: 세션 상세 데이터
- Redis 연결 끊김 시 메모리와 불일치 발생 가능
