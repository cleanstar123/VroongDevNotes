## 1. 선정산 배치 시간 변경

> [!info] 배경
> - 선정산 배치(`vrRiderAcctDailyIfJob`)의 실행 시간을 오후에서 오전으로 변경 요청
> - `/api/v1/migration/migrate` API에서 `tableName=vr_rider_acct_daily`로 전날 데이터 처리

### 변경 내역

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 실행 시간 | 14:10 | **11:10** |
| Cron 표현식 | `0 10 14 * * ?` | `0 10 11 * * ?` |
| 트리거 이름 | `vrRiderAcctDailyIfTrigger1410` | `vrRiderAcctDailyIfTrigger1110` |

### 수정 파일

| 파일 | 위치 | 변경 내용 |
|------|------|----------|
| `MigrationQuartzConfig.java` | vroong-3pl-platform-batch | 트리거 시간 변경 |

### 코드 변경

```java
// src/main/java/com/vroong/batch/config/quartz/MigrationQuartzConfig.java

@Bean
public Trigger vrRiderAcctDailyIfTrigger1110() {
  return TriggerBuilder.newTrigger()
      .forJob(vrRiderAcctDailyIfJobDetail())
      .withIdentity("vrRiderAcctDailyIfTrigger1110", "migrationGroup")
      .withDescription("오전 11:10 VR 라이더 계정 일일 정보 마이그레이션")
      .withSchedule(CronScheduleBuilder.cronSchedule("0 10 11 * * ?"))
      .build();
}
```

### DB 변경 완료

> [!warning] SCHED_NAME 주의
> - application.yml에는 `VroongBatchScheduler`로 설정되어 있으나, 실제 운영 DB에서는 `schedulerFactoryBean` 사용
> - 쿼리 작성 시 실제 DB 값 확인 필요

#### 실제 추가한 데이터

**QRTZ_JOB_DETAILS:**

**QRTZ_TRIGGERS:**

| SCHED_NAME | TRIGGER_NAME | TRIGGER_GROUP | JOB_NAME | TRIGGER_STATE | TRIGGER_TYPE |
|------------|--------------|---------------|----------|---------------|--------------|

**QRTZ_CRON_TRIGGERS:**

| SCHED_NAME | TRIGGER_NAME | TRIGGER_GROUP | CRON_EXPRESSION | TIME_ZONE_ID |

## 2. 자동 페이백(건당) 기능 - 달성페이백 계산 버그 수정

> [!info] 배경
> - 협력사 주정산 업로드 시 달성페이백(ACHIEVEMENT_PAYBACK)이 0으로 저장되는 문제
> - 포탈에서 페이백 설정(30원/건)을 했음에도 정산 시 반영되지 않음

### 발견된 문제들

#### 문제 1: AUTO_PAYBACK 컬럼 오류

> [!bug] 오류 메시지
> `Unknown column 'sawe1_0.AUTO_PAYBACK' in 'field list'`

- **원인**: SuppAcctWeeklyEntity에 `autoPayback` 필드가 있었으나 실제 DB 테이블에는 해당 컬럼 없음
- **해결**: 기존 `ACHIEVEMENT_PAYBACK` 컬럼을 사용하도록 변경

| 수정 파일 | 변경 내용 |
|----------|----------|
| `SuppAcctWeeklyEntity.java` | `autoPayback` 필드 제거 |
| `PartnerSettlement.java` | `autoPayback` 필드 제거 |
| `PartnerSettlementRepositoryImpl.java` | `autoPayback` 매핑 제거 |
| `PartnerSettlementUseCase.java` | `.achievementPayback(BigDecimal.ZERO).autoPayback(autoPayback)` → `.achievementPayback(autoPayback)` |

#### 문제 2: FEE_TYPE 불일치로 페이백 설정 조회 실패

- **원인**:
  - 저장 시: `FEE_TYPE = 'C'` (FEE_TYPE_HEADQUARTERS)
  - 조회 시: `FEE_TYPE = 'S'` (FEE_TYPE_SUPPLIER)
- **해결**: `calculateAutoPayback`에서 `companyFeesMap` (FEE_TYPE='C') 사용

| 수정 파일 | 변경 내용 |
|----------|----------|
| `PartnerSettlementUseCase.java` | `BatchData` 클래스에 `companyFeesMap` 필드 추가 |
| `PartnerSettlementUseCase.java` | `prepareBatchData()`에서 `FEE_TYPE_COMPANY`로 본사 수수료 조회 |
| `PartnerSettlementUseCase.java` | `calculateAutoPayback` 호출 시 `companyFeesMap` 사용 |

```java
// 변경 전
BigDecimal autoPayback = calculateAutoPayback(supplyId, manageFee, agg.deliveryCntMap,
        batchData.supplierFeesMap.getOrDefault(supplyId, new ArrayList<>()),
        batchData.registeredRiderIds);

// 변경 후
BigDecimal autoPayback = calculateAutoPayback(supplyId, maintncVal, agg.deliveryCntMap,
        batchData.companyFeesMap.getOrDefault(supplyId, new ArrayList<>()),
        batchData.registeredRiderIds);
```

#### 문제 3: 페이백 계산 조건 오류

- **원인**: `manageFee` (지점운영수수료) > 0 조건으로 체크
- **비즈니스 요구사항**: `maintncVal` (관리비) > 0 일 때 페이백 계산
- **해결**: 조건을 `maintncVal` 체크로 변경

| 항목 | 변경 전 | 변경 후 |
|-----|--------|--------|
| 조건 | `manageFee > 0` | `maintncVal > 0` |
| 의미 | 지점운영수수료 발생 시 | 관리비 발생 시 |

```java
// 변경 전
private BigDecimal calculateAutoPayback(String suppId, BigDecimal manageFee, ...) {
    if (manageFee == null || manageFee.compareTo(BigDecimal.ZERO) <= 0) {
        return BigDecimal.ZERO;
    }
    ...
}

// 변경 후
private BigDecimal calculateAutoPayback(String suppId, BigDecimal maintncVal, ...) {
    if (maintncVal == null || maintncVal.compareTo(BigDecimal.ZERO) <= 0) {
        return BigDecimal.ZERO;
    }
    ...
}
```

### 계산 공식

```

- 관리비(MAINTNC_VAL) > 0
- vr_fee_mst에 PAYBACK_USE_YN = 'Y', PAYBACK_AMT > 0 설정
- FEE_TYPE = 'C', SETTLEMENT_PERIOD = 'WEEKLY', FEE_TARGET = 'SRA'
```

### 페이백 관련

- **vr_fee_mst 수수료 유형**:
  - `FEE_TYPE = 'C'`: 본사 수수료 (본사 → 협력사 지급)
  - `FEE_TYPE = 'S'`: 협력사 수수료 (협력사 → 라이더)
- **페이백 지급 조건**: 관리비(MAINTNC_VAL) > 0 일 때만 계산
- **지급관리비 VAT포함 계산**: `(관리비 + 지점운영수수료 + 달성페이백 + 프로모션) × 110%`
