# 고액 입금 출금제한 기능 테스트 케이스

> 기능 개요: 가상계좌 입금 시 금액 기준으로 출금제한을 설정한다.
> - 100만원 미만 입금 → 당일 출금제한 (`WITHDRAWAL_YN = 'Y'`), 다음날 배치에서 자동 해제
> - 100만원 이상 입금 → 무기한 출금제한 (`WITHDRAWAL_YN = 'P'`), 라이더에게 팝업 공지 발송

---

## 테스트 환경 정보

| 항목 | 값 |
|------|----|
| 테스트 라이더 ID | `dudgh1050` |
| 가상계좌번호 (`IACCT_NO`) | `79032750536766` |
| VACS_AHST 스키마 | `vraccount` |

---

## 사전 조회 — BANK_CD 및 ORG_CD 확인

INSERT 전에 아래 두 값을 반드시 확인한다. `BANK_CD`는 VACS_AHST와 `vr_virtual_account_list`의 조인 키이며, 불일치 시 라이더 정보를 찾지 못해 출금제한이 설정되지 않는다.

```sql
-- 1. 테스트 라이더 가상계좌의 BANK_CD 확인
SELECT BANK_CD, ACCT_NO, ACCT_TYPE, SUPP_ID, RIDER_ID, ACCT_ST
FROM vr_virtual_account_list
WHERE ACCT_NO = '79032750536766'
  AND ACCT_ST = '1';
-- → BANK_CD 값을 아래 INSERT에 사용

-- 2. 기존 VACS_AHST 데이터에서 ORG_CD 확인 (실제 헥토 기관코드 사용)
SELECT ORG_CD, TR_CD, COUNT(*) AS CNT
FROM vraccount.VACS_AHST
GROUP BY ORG_CD, TR_CD
LIMIT 5;
-- → ORG_CD (8자리), TR_CD (4자리) 값을 아래 INSERT에 사용

-- 3. 테스트 라이더 현재 상태 확인
SELECT ri.RIDER_ID, ri.SUPP_ID, ri.WITHDRAWAL_YN, am.BAL_AMT
FROM vr_rider_info ri
LEFT JOIN vr_account_mst am
  ON ri.RIDER_ID = am.RIDER_ID
  AND ri.SUPP_ID = am.SUPP_ID
  AND am.ACCT_TARGET = 'R'
  AND am.SEQ_NO = (
    SELECT MAX(SEQ_NO) FROM vr_account_mst
    WHERE RIDER_ID = ri.RIDER_ID AND SUPP_ID = ri.SUPP_ID AND ACCT_TARGET = 'R'
  )
WHERE ri.RIDER_ID = 'dudgh1050';
```

## TC-01: 100만원 미만 입금 → 당일 출금제한 (Y)

### 목적
500,000원 입금 시 `WITHDRAWAL_YN='Y'`로 설정되고, 출금 시도 시 "다음날 다시 시도" 팝업이 표시되는지 확인

### Step 1 — VACS_AHST 테스트 데이터 삽입 (500,000원)

```sql
-- 위에서 조회한 BANK_CD, ORG_CD, TR_CD 값을 채워 넣을 것
INSERT INTO vraccount.VACS_AHST (
    ORG_CD,       -- 사전 조회한 기관코드 (8자리)
    TR_IL,        -- 오늘 날짜 (YYYYMMDD)
    TR_SI,        -- 거래시간 (HHMMSS)
    TR_CD,        -- 사전 조회한 거래종류코드 (4자리)
    BANK_CD,      -- 사전 조회한 가상계좌 은행코드 (3자리)
    IACCT_NO,     -- 테스트 라이더 가상계좌번호
    IACCT_NM,     -- 입금의뢰인명 (임의)
    TR_AMT,       -- 거래금액 (100만원 미만)
    MITA_AMT,
    TR_NO,        -- 거래고유번호 (7자리, ORG_CD+TR_IL 조합 내 유니크)
    INP_ST,       -- '1': 입금 처리 완료
    TRANSFER_YN   -- 'N': 미이관
) VALUES (
    'HECTO001',   -- ← 사전 조회 값으로 교체
    '20260702',   -- ← 오늘 날짜로 교체
    '120000',
    '1101',       -- ← 사전 조회 값으로 교체
    '{{BANK_CD}}',  -- ← 사전 조회 값으로 교체 (예: '011')
    '79032750536766',
    '테스트입금자',
    500000,
    0,
    'T000001',    -- ← 동일 ORG_CD+TR_IL에서 유니크한 7자리 값
    '1',
    'N'
);

-- 삽입 확인
SELECT * FROM vraccount.VACS_AHST ORDER BY SEQ_NO DESC LIMIT 3;
```

### Step 2 — 입금 처리 API 호출

```
POST /api/v1/hecto/deposit/process
Header: API-Secret: {{시크릿키}}
```

서버 로그에서 아래 메시지 확인:
```
INFO  - 라이더 입금 당일 출금제한 설정: suppId=..., riderId=dudgh1050
```

### Step 3 — DB 확인

```sql
-- 출금제한 Y 설정 확인
SELECT RIDER_ID, WITHDRAWAL_YN, UPD_DATE, UPD_USER
FROM vr_rider_info
WHERE RIDER_ID = 'dudgh1050';
-- 기대값: WITHDRAWAL_YN = 'Y'

-- 입금 내역 기록 확인
SELECT SEQ_NO, WD_TYPE, REQ_VAL, BAL_AMT, BRIEFS, INS_DATE
FROM vr_withdrl_list
WHERE RIDER_ID = 'dudgh1050'
ORDER BY INS_DATE DESC
LIMIT 3;
-- 기대값: WD_TYPE='D', BRIEFS='헥토파이낸셜 입금', REQ_VAL=500000

-- VACS_AHST 이관 처리 확인
SELECT SEQ_NO, TR_AMT, INP_ST, TRANSFER_YN
FROM vraccount.VACS_AHST
WHERE IACCT_NO = '79032750536766'
ORDER BY SEQ_NO DESC
LIMIT 3;
-- 기대값: TRANSFER_YN = 'Y' (처리 완료)

-- 공지가 생성되지 않았는지 확인 (100만원 미만이므로)
SELECT * FROM vr_notice
WHERE WRITER_ID = 'SYSTEM'
  AND TARGET_TYPE = 'R'
  AND INS_DATE >= NOW() - INTERVAL 5 MINUTE;
-- 기대값: 0건
```

### Step 4 — 앱 출금 시도

- 앱 로그인 → 출금 탭 → 금액 입력 → 출금하기 클릭
- **기대 팝업**: "출금 제한 안내" 모달 표시
  > 금일 입금내역이 있어 출금제한되었습니다. 다음날 다시 시도해주세요.

### 판정 기준

| 항목 | 기대값 |
|------|--------|
| `WITHDRAWAL_YN` | `Y` |
| `vr_withdrl_list` WD_TYPE | `D` (입금 기록 존재) |
| `VACS_AHST.TRANSFER_YN` | `Y` |
| `vr_notice` 공지 생성 | 없음 |
| 앱 팝업 메시지 | "금일 입금내역이 있어 출금제한되었습니다. 다음날 다시 시도해주세요." |

## TC-03: 배치 실행 후 Y는 해제, P는 유지

### 목적
배치 API 실행 시 `Y` 라이더만 `N`으로 해제되고, `P` 라이더는 변경되지 않는지 확인

### Step 1 — 두 상태를 DB에서 직접 설정

```sql
-- 테스트용 라이더를 Y 상태로 설정 (dudgh1050 사용)
UPDATE vr_rider_info
SET WITHDRAWAL_YN = 'Y', UPD_DATE = NOW(), UPD_USER = 'TEST'
WHERE RIDER_ID = 'dudgh1050';

-- 배치 해제 쿼리 조건 확인: BAL_AMT >= 0, WITHDRAWAL_YN = 'Y', 협력사 출금허용(N) 필요
SELECT am.RIDER_ID, am.BAL_AMT, ri.WITHDRAWAL_YN
FROM vr_account_mst am
JOIN vr_rider_info ri ON am.RIDER_ID = ri.RIDER_ID AND am.SUPPLY_ID = ri.SUPP_ID
WHERE am.RIDER_ID = 'dudgh1050'
  AND am.ACCT_TARGET = 'R'
ORDER BY am.SEQ_NO DESC
LIMIT 1;
-- BAL_AMT >= 0 이어야 배치 해제 대상이 됨
```

### Step 2 — 배치 출금제한 처리 API 호출

```
POST /api/v1/batch-withdrawal/withdrawal
Header: API-Secret: {{시크릿키}}
```

### Step 3 — 결과 확인 (Y → N)

```sql
SELECT RIDER_ID, WITHDRAWAL_YN, UPD_DATE
FROM vr_rider_info
WHERE RIDER_ID = 'dudgh1050';
-- 기대값: WITHDRAWAL_YN = 'N'
```

### Step 4 — P 유지 확인

```sql
-- P 상태로 직접 설정 후 배치 재실행
UPDATE vr_rider_info
SET WITHDRAWAL_YN = 'P', UPD_DATE = NOW(), UPD_USER = 'TEST'
WHERE RIDER_ID = 'dudgh1050';
```

배치 API 재호출 후:

```sql
SELECT RIDER_ID, WITHDRAWAL_YN
FROM vr_rider_info
WHERE RIDER_ID = 'dudgh1050';
-- 기대값: WITHDRAWAL_YN = 'P' (변경 없음)
```

### 판정 기준

| 상태 | 배치 후 |
|------|---------|
| `Y` (양수 잔액, 협력사 허용) | → `N` 해제 |
| `P` | → `P` 그대로 유지 |

## TC-04: P 상태 라이더에 100만원 미만 재입금 시 동작 확인

### 목적
`P` 상태인 라이더에 100만원 미만 입금이 들어오면 `Y`로 덮어씌워지는 현재 동작을 확인하고 개선 필요 여부 판단

### Step 1 — 라이더를 P 상태로 설정

```sql
UPDATE vr_rider_info
SET WITHDRAWAL_YN = 'P', UPD_DATE = NOW(), UPD_USER = 'TEST'
WHERE RIDER_ID = 'dudgh1050';
```

### Step 2 — VACS_AHST에 100만원 미만 입금 삽입 (300,000원)

    INP_ST, TRANSFER_YN
) VALUES (
    'HECTO001',      -- ← 사전 조회 값으로 교체
    '20260702',      -- ← 오늘 날짜로 교체
    '140000',
    '1101',          -- ← 사전 조회 값으로 교체
    '{{BANK_CD}}',   -- ← 사전 조회 값으로 교체
    '79032750536766',
    'P상태재입금테스트',

### Step 3 — 입금 처리 API 호출 후 확인

```sql

```

### 판정 기준 (버그 수정 완료)

> `updateWithdrawalYn()` WHERE 절에 `.and(vrRiderInfo.withdrawalYn.notIn("P"))` 조건 추가로 수정됨.
> P 상태인 라이더에게 100만원 미만 입금이 들어와도 UPDATE가 0건 실행되어 P가 유지된다.

| 항목 | 기대값 |
|------|--------|
| `WITHDRAWAL_YN` | `P` 그대로 유지 (UPDATE 0건) |
| 출금 시도 시 팝업 | "고액 입금으로 인해 출금제한되었습니다." (P 메시지 유지) |
