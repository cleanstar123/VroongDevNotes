# 프로모션 테스트 — 미수금 발생 & 다음 정산에서 해소

> 협력사: `DP2503071840`

---

## 목표

프로모션 PD 차감으로 잔액이 음수(미수금 발생)가 된 상태에서,
다음 주정산 업로드 시 TRAN_AMT가 양수라면 UNPAID_AMT가 해소되는지 확인.

---

## 미수금 발생 원리

```
PD 생성 시:
  newBalance = previousBalance + (-TOTAL_PROMO_COST)
  unpaidAmt  = newBalance < 0 ? newBalance : 0

다음 SW(주정산 수령) 생성 시:
  newBalance = previousBalance(음수) + ACCT_VAL(양수)
  unpaidAmt  = newBalance < 0 ? newBalance : 0
                                ↑ 양수가 되면 unpaidAmt = 0 → 미수금 해소
```

## STEP 1 — 미수금 상태 만들기

> 기존 PD 방식(HELD apply)으로 미수금을 만들려면 이전 잔액이 TOTAL_PROMO_COST보다 작아야 합니다.
> 가장 간단한 방법: **vr_account_mst에 음수 PD 레코드를 직접 삽입**합니다.

### 방법 A — 직접 INSERT (권장)

```sql

-- previousBalance = 0 가정, TOTAL_PROMO_COST = 310,000원 설정
INSERT INTO vr_account_mst
  (ACCT_TYPE, ACCT_DATE, ACCT_TARGET, SUPPLY_ID, RIDER_ID,
   BAL_AMT, TRAN_AMT, UNPAID_AMT, INS_USER, INS_DATE)

  ('PD', '20260623', 'S', 'DP2503071840', '',
   -310000, -310000, -310000, 'TEST', NOW());
```

삽입 후 상태 확인:

```sql
SELECT SEQ_NO, ACCT_TYPE, ACCT_DATE, BAL_AMT, TRAN_AMT, UNPAID_AMT
```

### 방법 B — PENDING_DEPOSIT + FORCED (실제 흐름)

1. PENDING_DEPOSIT 상태에서 관리자 강제 지급 API 호출 (`POST /force`)
2. `forceProcess` 실행 → SW + PD 생성 (협력사), FORCED로 전환 ⭐ (2026-06-30 구현 완료)
3. `applyRidersForSuppId` 실행 → 라이더 RW 생성 (promotion 복원 포함), `vr_promo_rider_result` 삭제
4. 이 흐름으로도 미수금 상태 테스트 가능 (vr_account_mst에 실제 PD 레코드 생성됨)

### STEP 2-1. 라이더 주정산 엑셀 업로드

- 기존과 동일하게 업로드

### STEP 2-2. 협력사 주정산 엑셀 업로드

업로드 후 SW 엔트리 확인:

```sql
SELECT
    ACCT_TYPE,
    ACCT_DATE,
    BAL_AMT,
    TRAN_AMT,
    UNPAID_AMT
FROM vr_account_mst

  AND ACCT_TARGET = 'S'
  AND ACCT_TYPE IN ('SW', 'SR')
ORDER BY SEQ_NO DESC LIMIT 1;
```

**기대 결과:**

| 항목 | 계산식 | 기대값 |
|------|--------|--------|
| `previousBalance` | vr_account_mst 최신 BAL_AMT | -310,000원 |
| `TRAN_AMT` | ACCT_VAL (이번 주 주정산) | +X원 (양수) |
| `BAL_AMT` | -310,000 + X | X - 310,000원 |
| `UNPAID_AMT` | BAL_AMT ≥ 0 → **0원** | **0원 (해소 확인!)** |

> ⚠️ ACCT_VAL(X)이 310,000원보다 작으면 BAL_AMT가 여전히 음수 → UNPAID_AMT = BAL_AMT (미해소)

## 미수금 미해소 케이스도 확인

ACCT_VAL이 미수금보다 작은 경우도 테스트합니다.

```
PD   (20260623): BAL_AMT = -310,000 / UNPAID_AMT = -310,000
SW   (20260630): ACCT_VAL = 100,000 → BAL_AMT = -210,000 / UNPAID_AMT = -210,000  ← 부분 해소
```

이 경우 다다음 주 정산에서 추가 해소가 이루어지는지 확인합니다.

## 기대 결과 요약

| 조건 | TRAN_AMT | BAL_AMT | UNPAID_AMT | 미수금 상태 |
|------|:--------:|:-------:|:----------:|:-----------:|
| ACCT_VAL > 310,000 | +양수 | 양수 | 0 | ✅ 완전 해소 |
| ACCT_VAL = 310,000 | +310,000 | 0 | 0 | ✅ 완전 해소 |
| ACCT_VAL < 310,000 | +양수 | 음수 | BAL_AMT | ⚠️ 부분 해소 |
