### ⚠️ 강제진행(Test A) 완전 테스트를 위한 추가 사전 작업

강제진행 시 `vr_promo_rider_result` 데이터가 있어야 라이더 promotion이 복원됩니다.
**상태 초기화(UPDATE만)로는 vr_promo_rider_result가 없으므로 promotion 복원이 스킵됩니다.**

완전한 흐름 테스트가 필요하면 아래 중 하나를 선택:

**방법 1 (권장)**: 처음부터 업로드 → 자동으로 vr_promo_rider_result 생성
1. 기존 데이터 삭제
2. 라이더 주정산 업로드
3. 협력사 주정산 업로드 → PENDING_DEPOSIT 결정 시 자동으로 `savePromoAndClearForPendingDeposit` 호출

**방법 2**: 수동 vr_promo_rider_result 삽입 (빠른 테스트용)
```sql
-- 샘플 데이터 (실제 라이더 PROMOTION 금액을 STEP 0에서 확인 후 삽입)
INSERT INTO vr_promo_rider_result (SUPP_ID, RIDER_ID, ACCT_ST_DATE, ACCT_ED_DATE, PROMOTION_AMT, INS_USER)
SELECT 'DP2503071840', RIDER_ID, '2026-06-17', '2026-06-23', PROMOTION, 'TEST'
FROM vr_rider_acct_weekly
WHERE SUPPLY_ID = 'DP2503071840'
  AND ACCT_ST_DATE = '20260617'
  AND PROMOTION IS NOT NULL
  AND PROMOTION > 0;
-- 실행 전 PROMOTION이 null이면 데이터 없음 → 방법 1 사용
```

       → vr_promo_rider_result 삭제 (promotion 포기 확정)
       → PROCESSED

PENDING_DEPOSIT
  └─ Apply 버튼: 해당 협력사 라이더 vr_account_mst 미생성 (차단)
