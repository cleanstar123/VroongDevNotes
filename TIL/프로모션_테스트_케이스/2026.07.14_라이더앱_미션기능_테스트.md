# 라이더 앱 미션 기능 테스트

> 신규 개발 기능: 홈화면 플로팅 팝업 + 미션 페이지  
> API: `GET /api/v1/mob/mission/list`  
> 협력사: `DP2503071840` | 기준일: `2026-07-14 (월)`

---

## 테스트 전 사전 확인

### API 응답 구조

```json
{
  "missions": [
    {
      "seqNo": 1,
      "title": "미션 제목",
      "missionType": "FIXED | OVERAGE | FULL_UNIT",
      "periodType": "DAILY | WEEKLY",
      "targetCnt": 30,
      "rewardAmt": 10000,
      "maxCnt": 0,
      "startDate": "2026-07-14",
      "endDate": "2026-07-20",
      "description": "설명",
      "riderStatus": "UPCOMING | IN_PROGRESS | COMPLETED | SETTLEMENT_EXPECTED | SETTLED",
      "expectedPaymentDate": "2026-07-27"
    }
  ],
  "hasActiveMission": true,
  "hasPendingMission": false,
  "latestEndDate": "2026-07-20"
}
```

### 플로팅 버튼(FAB) 표시 규칙 요약

| 상태              | hasActiveMission | hasPendingMission | FAB 표시 | 배지(빨간 점) |
| --------------- | :--------------: | :---------------: | :----: | :------: |
| 진행중/예정 미션 존재    |      `true`      |        any        |  ✅ 표시  |   ✅ 있음   |
| 완료/정산예정 만 존재    |     `false`      |      `true`       | ✅ 반투명  |   ❌ 없음   |
| 전부 정산완료 (1일 이내) |     `false`      |      `false`      | ✅ 반투명  |   ❌ 없음   |
| 전부 정산완료 (1일 경과) |     `false`      |      `false`      | ❌ 미노출  |    —     |
| 미션 없음           |     `false`      |      `false`      | ❌ 미노출  |    —     |

### 라이더 준비 (실제 라이더 ID로 교체)

```sql
-- 테스트에 사용할 라이더 확인 (DP2503071840 소속)
SELECT RIDER_ID, MAX(ACCT_ST_DATE) AS last_acct
FROM vr_rider_acct_weekly
WHERE SUPPLY_ID = 'DP2503071840'
GROUP BY RIDER_ID
ORDER BY last_acct DESC
LIMIT 10;
-- → 사용할 라이더 ID를 아래 {RIDER_ID}에 기입
```

> **목적**: `vr_promo_rider_result` 존재 + `vr_promo_supp_settlement.HOLD_STATUS ≠ PROCESSED/FORCED`

```sql
SELECT SEQ_NO FROM vr_promo_mission
WHERE SUPP_ID = 'DP2503071840' AND TITLE = '[TEST] TC04-정산예정';
```

## TC-05: 정산완료 미션 — `SETTLED` 표시 + 1일 이내 FAB 유지

> **목적**: `vr_promo_supp_settlement.HOLD_STATUS = 'PROCESSED'`  
> → `riderStatus = SETTLED`, `hasActiveMission = false`, `hasPendingMission = false`  
> → `latestEndDate` 기준 1일 이내이면 FAB 반투명 유지, 이후 완전 미노출

### STEP 1 — 미션 + 정산완료 데이터 생성

```sql
-- 미션 등록 (지난 주)
INSERT INTO vr_promo_mission
  (SUPP_ID, TITLE, MISSION_TYPE, PERIOD_TYPE, TARGET_CNT, REWARD_AMT, MAX_CNT,
   START_DATE, END_DATE, DESCRIPTION, TARGET_TYPE, STATUS_CODE, INS_DATE, INS_USER)
VALUES
  ('DP2503071840', '[TEST] TC05-정산완료', 'FIXED', 'WEEKLY', 30, 10000, 0,
   '2026-07-07', '2026-07-13', '정산완료 SETTLED 테스트', 'ALL', 'ACTIVE', NOW(), 'TEST');

-- 정산 결과 및 완료 상태 생성
INSERT INTO vr_promo_rider_result
  (SUPP_ID, RIDER_ID, ACCT_ST_DATE, ACCT_ED_DATE, PROMOTION_AMT, INS_DATE, INS_USER)

  ('DP2503071840', '{RIDER_ID}', '2026-07-07', '2026-07-13', 10000, NOW(), 'TEST')

  (SUPP_ID, ACCT_ST_DATE, ACCT_ED_DATE, TOTAL_PROMO_COST, HOLD_STATUS, INS_DATE, INS_USER)
```

### STEP 2 — API 기대 응답

```json
{
  "missions": [
    {
      "title": "[TEST] TC05-정산완료",

      "startDate": "2026-07-07",
```

### STEP 3 — 1일 경과 시나리오 비교

> `latestEndDate = 2026-07-13` → `2026-07-13 + 1일 = 2026-07-14`

| 테스트 시점 | 기대 FAB 동작 |
|-------------|--------------|
| 2026-07-14 (오늘, 1일 이내) | ✅ 반투명 표시 (배지 없음) |
| 2026-07-15 이후 (1일 경과) | ❌ FAB 완전 미노출 |

> **1일 경과 시뮬레이션**: 브라우저 날짜를 2026-07-15로 변경하거나, `latestEndDate`를 2026-07-12로 조작하여 확인.

```sql
-- latestEndDate를 어제로 바꾸어 1일 경과 테스트 (직접 미션 날짜 수정)
UPDATE vr_promo_mission
SET END_DATE = '2026-07-12'
WHERE SUPP_ID = 'DP2503071840' AND TITLE = '[TEST] TC05-정산완료';
-- API 재호출 시 latestEndDate='2026-07-12', 오늘=2026-07-14 > 2026-07-13 → FAB 미노출 기대
```

| FAB 배지 | ❌ 없음 | — |

| 미션 카드 불투명도 | ~65% (card-settled) | — |

## TC-06: 특정 라이더 대상 미션 (TARGET_TYPE = SPECIFIC) — 대상/비대상 필터링

> **목적**: `vr_promo_mission_rider`에 등록된 라이더에게만 미션 노출  
> 비대상 라이더: API 응답에 해당 미션 미포함 → FAB 미노출

### 테스트 라이더 설정

| 라이더 ID | 대상 여부 |
|-----------|:---:|
| `{RIDER_A}` | ✅ 대상 |
| `{RIDER_B}` | ❌ 비대상 |

### STEP 1 — 미션 등록 (SPECIFIC)

```sql
INSERT INTO vr_promo_mission
  (SUPP_ID, TITLE, MISSION_TYPE, PERIOD_TYPE, TARGET_CNT, REWARD_AMT, MAX_CNT,
   START_DATE, END_DATE, DESCRIPTION, TARGET_TYPE, STATUS_CODE, INS_DATE, INS_USER)
VALUES
  ('DP2503071840', '[TEST] TC06-특정라이더', 'FIXED', 'WEEKLY', 30, 10000, 0,
   '2026-07-14', '2026-07-20', '특정 라이더 필터링 테스트', 'SPECIFIC', 'ACTIVE', NOW(), 'TEST');

SELECT SEQ_NO FROM vr_promo_mission
WHERE SUPP_ID = 'DP2503071840' AND TITLE = '[TEST] TC06-특정라이더';

-- 대상 라이더 등록 (SEQ_NO 교체 필요)

-- 등록 확인
SELECT * FROM vr_promo_mission_rider WHERE MISSION_SEQ_NO = {SEQ_NO};
```

### STEP 2 — {RIDER_A} API 호출 기대 응답

```json
{
  "missions": [

      "title": "[TEST] TC06-특정라이더",
```

### STEP 3 — {RIDER_B} API 호출 기대 응답

```sql
-- 백엔드 로직과 동일한 계산을 SQL로 검증
SELECT
    TITLE,
    END_DATE,
    DAYNAME(END_DATE) AS 종료요일,
    -- 다음 월요일 계산
    DATE_ADD(END_DATE, INTERVAL
        CASE DAYOFWEEK(END_DATE)
            WHEN 2 THEN 7  -- 월요일이면 +7
            WHEN 3 THEN 6  -- 화요일
            WHEN 4 THEN 5  -- 수요일
            WHEN 5 THEN 4  -- 목요일
            WHEN 6 THEN 3  -- 금요일
            WHEN 7 THEN 2  -- 토요일
            WHEN 1 THEN 1  -- 일요일
        END DAY) AS expected_payment_date
```
