# 프로모션 테스트 — PENDING_DEPOSIT & 공지사항

> 협력사: `DP2503071840` | 정산 기간: `20260617 ~ 20260623`

---

## 발생 조건

```
협력사 주정산비용(PARTNER_ACCT_VAL) < 미션 총비용(TOTAL_PROMO_COST)
→ HOLD_STATUS = PENDING_DEPOSIT
→ vr_notice + vr_notice_target 레코드 자동 생성
→ vr_promo_supp_settlement.NOTICE_ID 채워짐
```

---

## STEP 0 — 기존 PARTNER_ACCT_VAL 확인

PENDING_DEPOSIT을 만들려면 TOTAL_PROMO_COST > PARTNER_ACCT_VAL 이어야 합니다.

```sql
SELECT SUPPLY_ID, ACCT_ST_DATE, ACCT_VAL
FROM vr_supp_acct_weekly
WHERE SUPPLY_ID = 'DP2503071840' AND ACCT_ST_DATE = '20260617';
```

> 이 테스트에서는 ACCT_VAL ≈ 1,801,793 기준으로 이를 초과하는 TOTAL_PROMO_COST를 설정합니다.

---

달성 라이더 31명 × 100,000원 = **3,100,000원** > ACCT_VAL(≈1,801,793) → PENDING_DEPOSIT 발생

```sql
INSERT INTO vr_promo_mission
  (SUPP_ID, TITLE, MISSION_TYPE, PERIOD_TYPE, TARGET_CNT, REWARD_AMT, MAX_CNT,
   START_DATE, END_DATE, DESCRIPTION, TARGET_TYPE, STATUS_CODE, INS_DATE, INS_USER)
VALUES
  ('DP2503071840', '[TEST] PENDING_DEPOSIT', 'FIXED', 'WEEKLY', 30, 100000, 0,
   '2026-06-17', '2026-06-23', 'PENDING_DEPOSIT 공지 테스트', 'ALL', 'ACTIVE', NOW(), 'TEST');

SELECT SEQ_NO, TITLE, REWARD_AMT FROM vr_promo_mission
WHERE SUPP_ID = 'DP2503071840' AND TITLE = '[TEST] PENDING_DEPOSIT';
```

**기대 TOTAL_PROMO_COST**: 달성 31명 × 100,000원 = **3,100,000원**

## STEP 3 — 협력사 주정산 엑셀 업로드

### 검증 쿼리 ②: PENDING_DEPOSIT 전이 확인


## STEP 3.5 — PENDING_DEPOSIT 결정 시 자동 실행된 동작 확인 ⭐ 신규

> 협력사 업로드 완료 시 PENDING_DEPOSIT으로 결정되면 `savePromoAndClearForPendingDeposit`이 자동 호출됩니다.
> 라이더가 받지 못할 프로모션 금액을 정산내역에서 미리 제거합니다.

### 검증 쿼리 ③-A: 라이더 PROMOTION null 클리어 확인

```sql
SELECT
    RIDER_ID,
    PROMOTION,           -- ← NULL이어야 함 (자동 클리어됨)
    TAXABLE_DELIVERY_VAL -- ← PROMOTION=0 기준으로 재계산된 값
FROM vr_rider_acct_weekly
WHERE SUPPLY_ID = 'DP2503071840'
  AND ACCT_ST_DATE = '20260617'
ORDER BY RIDER_ID
LIMIT 10;
-- PROMOTION이 null이어야 함 (강제진행이 아닌 이상 프로모션 미지급)
```

### 검증 쿼리 ③-B: vr_promo_rider_result 저장 확인

```sql
SELECT
    RIDER_ID,
    PROMOTION_AMT,       -- ← STEP 2에서 계산한 각 라이더 지급액
    ACCT_ST_DATE,
    ACCT_ED_DATE
FROM vr_promo_rider_result
WHERE SUPP_ID = 'DP2503071840'
  AND ACCT_ST_DATE = '2026-06-17'
ORDER BY PROMOTION_AMT DESC;
-- 프로모션이 0이 아닌 라이더들의 금액이 저장되어 있어야 함
-- 총 SUM(PROMOTION_AMT) = 3,100,000원 이어야 함
```

### 검증 쿼리 ③: 공지사항 내용 확인

> 위 쿼리에서 NOTICE_ID 값을 확인 후 아래 쿼리에 넣으세요.

```sql
-- 공지 내용 확인
SELECT
    NOTICE_ID,
    TITLE,
    CONTENT,
    TARGET_TYPE,
    TARGET_SCOPE,
    POPUP_YN,
    URGENT_YN,
    USE_YN
FROM vr_notice
WHERE NOTICE_ID = {NOTICE_ID};
```

**기대 내용:**
- `TITLE`: `[프로모션] 2026-06-17 ~ 2026-06-23 기간 입금 요청`
- `CONTENT`: 프로모션 비용(3,100,000원)이 주정산비용(≈1,801,793원)을 초과, 차액 입금 요청
- `POPUP_YN` = `Y`, `URGENT_YN` = `Y`, `USE_YN` = `Y`

### 검증 쿼리 ④: 공지 대상 확인

```sql
SELECT NOTICE_ID, TARGET_ID, TARGET_TYPE
FROM vr_notice_target
WHERE NOTICE_ID = {NOTICE_ID};
-- TARGET_ID = 'DP2503071840', TARGET_TYPE = 'S'
```

## STEP 5 — Apply 시 PENDING_DEPOSIT 건 처리 확인

Apply 버튼 클릭 시 PENDING_DEPOSIT 협력사는 **skip** 되어야 합니다.

```sql
-- Apply 후에도 여전히 PENDING_DEPOSIT 상태여야 함
SELECT HOLD_STATUS FROM vr_promo_supp_settlement
WHERE SUPP_ID = 'DP2503071840';
-- 'PENDING_DEPOSIT' (PROCESSED로 변경되면 안 됨)

-- PD 레코드가 생성되지 않아야 함
SELECT COUNT(*) FROM vr_account_mst
WHERE SUPPLY_ID = 'DP2503071840' AND ACCT_TYPE = 'PD'
  AND ACCT_DATE = '20260623';
-- 0 이어야 함
```
