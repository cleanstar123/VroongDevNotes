# 프로모션 테스트 — OVERAGE (초과분 단가 지급)

> 협력사: `DP2503071840` | 정산 기간: `20260617 ~ 20260623`

---

## 계산 공식

```
목표 건수(targetCnt) 이상 배달 시:
  초과분 = 실제 배달건수 - targetCnt
  (maxCnt > 0 이면) 초과분 = min(초과분, maxCnt)
  지급액 = 초과분 × rewardAmt

목표 미달 시: 0원
```

> **시나리오 2**: DP2503071840 → HELD, DP2603239509 → PENDING_DEPOSIT (REWARD_AMT=2,000 사용 시)

두 협력사 모두 협력사 주정산 업로드 후 확인합니다.

```sql
-- 두 협력사 상태 비교
SELECT SUPP_ID, TOTAL_PROMO_COST, PARTNER_ACCT_VAL, HOLD_STATUS, NOTICE_ID
FROM vr_promo_supp_settlement
WHERE SUPP_ID IN ('DP2503071840', 'DP2603239509')
  AND ACCT_ST_DATE = '2026-06-17';
-- 기대: DP2503071840 = HELD, DP2603239509 = PENDING_DEPOSIT

-- PENDING_DEPOSIT인 DP2603239509 라이더 PROMOTION 자동 클리어 확인
SELECT SUPPLY_ID, COUNT(*) AS total_riders,
       SUM(CASE WHEN PROMOTION IS NULL THEN 1 ELSE 0 END) AS null_promo_cnt
FROM vr_rider_acct_weekly
WHERE SUPPLY_ID IN ('DP2503071840', 'DP2603239509')
  AND ACCT_ST_DATE = '20260617'
GROUP BY SUPPLY_ID;
-- DP2503071840: null_promo_cnt = 0 (HELD이므로 PROMOTION 유지)
-- DP2603239509: null_promo_cnt = total_riders (PENDING_DEPOSIT이므로 전부 null)

-- vr_promo_rider_result 격리 확인 (PENDING_DEPOSIT인 DP2603239509만 존재)
SELECT SUPP_ID, COUNT(*) AS cnt, SUM(PROMOTION_AMT) AS total
FROM vr_promo_rider_result
WHERE SUPP_ID IN ('DP2503071840', 'DP2603239509')
  AND ACCT_ST_DATE = '2026-06-17'
GROUP BY SUPP_ID;
-- DP2503071840: 없음 (HELD이므로 저장 안 됨)
-- DP2603239509: N건 (PENDING_DEPOSIT이므로 자동 저장)
```

### STEP E — Apply 혼합 상태 독립 처리 검증

Apply 버튼 클릭 시 HELD는 처리되고 PENDING_DEPOSIT은 차단됩니다.

```sql
-- Apply 후 협력사 account_mst 확인
SELECT SUPPLY_ID, ACCT_TYPE, COUNT(*) AS cnt, SUM(TRAN_AMT) AS total_tran
FROM vr_account_mst
WHERE SUPPLY_ID IN ('DP2503071840', 'DP2603239509')
  AND ACCT_TARGET = 'S' AND ACCT_DATE = '20260623'
GROUP BY SUPPLY_ID, ACCT_TYPE;
-- DP2503071840: PD 1건 (HELD → 정상 차감)
-- DP2603239509: 없음 (PENDING_DEPOSIT → 차단)

-- Apply 후 라이더 RW 확인
SELECT SUPPLY_ID, COUNT(*) AS rider_rw_cnt
FROM vr_account_mst
WHERE SUPPLY_ID IN ('DP2503071840', 'DP2603239509')
  AND ACCT_TARGET = 'R' AND ACCT_TYPE = 'RW' AND ACCT_DATE = '20260623'
GROUP BY SUPPLY_ID;
-- DP2503071840: N건 (정상 지급)
-- DP2603239509: 0건 (차단)
```
