## 1. Vpay란?

현재 시스템 내에서 **Vpay**는 라이더 대상 전자지갑/선정산 지급 플랫폼으로, 아래 두 축으로 구성된다.

| 구성 | 설명 |
|------|------|
| **Portal (관리자 웹)** | 협력사·라이더 정산 관리, Vpay 가입 라이더 현황 조회, 엑셀 업로드/다운로드 |
| **Mobile API** | 라이더 앱 대상 API — 잔액 조회, 출금, 배달 이력, 공제 내역, 마이페이지 등 |

**Vpay 가입 기준**: `vr_user_info.USE_YN = 'Y'` 인 라이더가 Vpay 가입자로 판별됨

## 2. Vpay 관련 주요 컴포넌트 현황

### 2-1. 외부 연동

| 연동 대상                        | 역할                                                  | 현재 설정 위치                                 |
| ---------------------------- | --------------------------------------------------- | ---------------------------------------- |
| **헥토파이낸셜 (Hecto Financial)** | 가상계좌 입금 처리, 출금(이체) 처리                               | `application-dev.yml` → `vroong.hecto.*` |

| 테이블 | 역할 |
|--------|------|

| `vr_withdrl_list` | 출금 내역 |

| `vr_supp_acct_weekly` | 협력사 주정산 |

### 2-3. 모바일 API 엔드포인트 (라이더 앱)

```

GET  /api/v1/mob/delivery/history    — 배달 이력 조회
GET  /api/v1/mob/deduct              — 공제 내역 조회
GET  /api/v1/mob/mypage              — 마이페이지
GET  /api/v1/mob/supplier-settlement — 협력사 정산 조회 (앱용)
POST /api/v1/hecto/deposit           — 헥토 입금 내역 이관 처리 (트리거용)
```

## 3. 이관 시 발생하는 작업 항목

### 🔴 필수 — 인프라 / 자격증명 이전

#### 3-1. 헥토파이낸셜 계약 이전

**현재 상황**
- 헥토파이낸셜 계약은 부릉(또는 개발사) 명의로 체결된 상태
- 상인 ID(`HECTO_MCHT_ID`), 암호화키(`HECTO_ENCRYPTION_KEY`) 사용 중
- Hecto API는 **IP 화이트리스트** 제한이 있음 → VrAccountWeb 서버 IP(3.36.153.113) 등록 상태

**필요한 작업**
- [ ] 헥토파이낸셜과 **고객사 명의로 신규 계약 체결** (또는 계약 명의 변경)
- [ ] 고객사 계정으로 새 `MCHT_ID`, `ENCRYPTION_KEY` 발급
- [ ] 고객사 서버 IP를 헥토 화이트리스트에 등록
- [ ] 릴레이 서버(VrAccountWeb)를 고객사 인프라로 이전하거나, 신규 릴레이 구성
- [ ] `vroong.hecto.*` 환경변수를 고객사 신규 자격증명으로 교체

> **논의 필요**: 기존 가상계좌(vr_virtual_account_list)와 잔액 데이터를 신규 계약으로 마이그레이션 가능한지 헥토파이낸셜 측에 확인 필요

---

#### 3-2. 이메일(SMTP) 서버 이전

**현재 상황**
- Gmail SMTP(`cleanstar1212@gmail.com`)로 라이더 Vpay 가입 인증 메일 발송 중
- 이메일 브랜딩: `"Vpay"`, `"VROONG RIDERS"` 로고 포함

**필요한 작업**
- [ ] 고객사 도메인 기반 발신 이메일 주소로 변경 (예: `noreply@고객사도메인.com`)
- [ ] SMTP 서버 구성 또는 AWS SES 등 메일 서비스 계약
- [ ] 이메일 HTML 템플릿의 브랜딩(로고, 문구) 고객사 브랜드로 교체
- [ ] `spring.mail.*` 환경변수 변경

#### 3-3. DB 이전 / 접근권한 이전

**현재 상황**
- AWS RDS (`vraccountdb.c1acmo8sqyvw.ap-northeast-2.rds.amazonaws.com`) 사용 중

**필요한 작업**
- [ ] 고객사 AWS 계정(또는 자체 DB)으로 RDS 이전

## 6. 논의가 필요한 핵심 질문

1. **헥토파이낸셜 계약은 어떻게 처리할 것인가?**
   - 명의 이전 vs 신규 계약
   - 기존 가상계좌/잔액 데이터 연속성 확보 방법

2. **릴레이 서버(VrAccountWeb)를 고객사 인프라에서 운영할 수 있는가?**
   - 고객사 서버 IP를 헥토 화이트리스트에 등록 가능한지

3. **모바일 앱 배포는 누가 담당하는가?**
   - 앱스토어/플레이스토어 계정 이전 여부
   - 기존 라이더의 앱 강제 업데이트 시점

4. **6월 전환 시점에 데이터 단절 없이 이관할 수 있는가?**
   - 정산 이력, 잔액 원장 데이터의 continuity 보장 방법
   - 이관 당일 서비스 중단 시간(Downtime) 허용 여부

5. **이관 후 장애 발생 시 롤백 계획은?**
   - 일정 기간 구시스템 병행 유지 가능 여부
