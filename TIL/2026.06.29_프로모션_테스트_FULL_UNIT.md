# 프로모션 테스트 — FULL_UNIT (목표 달성 전체단가 지급)

> 협력사: `DP2503071840` | 정산 기간: `20260617 ~ 20260623`

---

## 계산 공식

```
목표 건수(targetCnt) 이상 배달 시:
  지급액 = min(실제 배달건수, maxCnt) × rewardAmt

maxCnt = 0 이면 상한 없음:
  지급액 = 실제 배달건수 × rewardAmt

목표 미달 시: 0원

※ OVERAGE와의 차이: OVERAGE는 목표 초과분에만 단가 적용,
   FULL_UNIT는 전체 배달건수(또는 maxCnt까지)에 단가 적용.
```

## STEP 3 — 라이더 주정산 엑셀 업로드

1. 20260617~20260623 기간 라이더 주정산 엑셀 업로드
2. 업로드 완료 후 아래 쿼리로 결과 확인

### 검증 쿼리 ①: 라이더별 PROMOTION 컬럼

```sql
SELECT
    RIDER_ID,
    DELIVERY_CNT,
    DELIVERY_VAL,
    ADD_VAL,
    PROMOTION,                -- ← 프로모션 지급액 (STEP 2 계산값과 일치 여부 확인)
    TAXABLE_DELIVERY_VAL,     -- ← 과세배달료 (PROMOTION 포함됨)
    TOT_ACCT_VAL
FROM vr_rider_acct_weekly
WHERE SUPPLY_ID = 'DP2503071840'
  AND ACCT_ST_DATE = '20260617'
ORDER BY PROMOTION DESC;
```

**확인 포인트:**
- `DELIVERY_CNT >= 30` 인 라이더: `PROMOTION = min(DELIVERY_CNT, 50) × 200`
- `DELIVERY_CNT < 30` 인 라이더: `PROMOTION = 0`
- `TAXABLE_DELIVERY_VAL` = `(DELIVERY_VAL + ADD_VAL + PROMOTION - TOTAL_MANAGE_FEE)` 를 10원 단위 내림한 값인지

### 검증 쿼리 ②: promo_supp_settlement PENDING 상태

```sql
SELECT
    SUPP_ID,
    ACCT_ST_DATE,
    ACCT_ED_DATE,
    TOTAL_PROMO_COST,   -- ← STEP 2 합계와 일치 여부
    HOLD_STATUS         -- ← 'PENDING' 이어야 함
FROM vr_promo_supp_settlement
WHERE SUPP_ID = 'DP2503071840';
```

```sql
    SUPPLY_ID,

WHERE SUPPLY_ID = 'DP2503071840'
  AND ACCT_ST_DATE = '20260617';
```

### 검증 쿼리 ③: HELD vs PENDING_DEPOSIT 상태 전이

```sql
SELECT
    SUPP_ID,
    TOTAL_PROMO_COST,
    PARTNER_ACCT_VAL,
    HOLD_STATUS,    -- 'HELD' 또는 'PENDING_DEPOSIT'
    NOTICE_ID       -- PENDING_DEPOSIT 시 공지 ID 존재
FROM vr_promo_supp_settlement
WHERE SUPP_ID = 'DP2503071840';
```

**판단 기준:**
- `PARTNER_ACCT_VAL >= TOTAL_PROMO_COST` → `HOLD_STATUS = HELD`
- `PARTNER_ACCT_VAL < TOTAL_PROMO_COST` → `HOLD_STATUS = PENDING_DEPOSIT`, `NOTICE_ID` 존재

**FULL_UNIT 미션 특이사항:**
- `maxCnt = 0`이면 건수에 비례해 총비용이 커짐 → PENDING_DEPOSIT 가능성 높음
- PENDING_DEPOSIT 확인 시: `rewardAmt`를 높게 설정하거나 `maxCnt = 0`으로 테스트
- HELD 확인 시: `maxCnt`를 낮게 설정 (totalPromoCost 최대값 제한)

## 기대 결과 요약

| 검증 항목 | 기대값 |
|-----------|--------|
| 달성 라이더 `PROMOTION` | `min(DELIVERY_CNT, MAX_CNT) × REWARD_AMT` |
| 미달 라이더 `PROMOTION` | 0원 |
| `TAXABLE_DELIVERY_VAL` | 배달료+추가지급+프로모션-운영수수료 (10원 내림) |
| `TOTAL_PROMO_COST` | 라이더 PROMOTION 합계와 일치 |
| 업로드 후 `HOLD_STATUS` | HELD 또는 PENDING_DEPOSIT |
| Apply 후 `TRAN_AMT` (PD) | -TOTAL_PROMO_COST |
| Apply 후 `HOLD_STATUS` | PROCESSED |
