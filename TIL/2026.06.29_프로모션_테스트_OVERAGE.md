# 프로모션 테스트 — OVERAGE (초과분 단가 지급)

> 협력사: `DP2503071840` | 정산 기간: `20260617 ~ 20260623`

---

## 계산 공식

```
목표 건수(targetCnt) 이상 배달 시:
  초과분 = 실제 배달건수 - targetCnt
  (maxCnt > 0 이면) 초과분 = min(초과분, maxCnt)
  지급액 = 초과분 × rewardAmt

목표 미달 시: 0원
```

---

## STEP 0 — 사전 확인: 실제 배달건수 조회

테스트 전 라이더별 실제 배달건수를 확인해서 기대값 계산에 사용합니다.

```sql
SELECT
    RIDER_ID,
    SUM(FINISH_CNT) AS total_finish_cnt
FROM vr_delivery_list
WHERE SUPPLY_ID = 'DP2503071840'
  AND DELIVERY_DATE BETWEEN '20260617' AND '20260623'
  AND FINISH_CNT > 0
GROUP BY RIDER_ID
ORDER BY total_finish_cnt DESC;
```

## STEP 2 — 기대값 계산

STEP 0에서 조회한 라이더별 건수를 이 표에 채워두세요.

| 라이더 ID | 실제 배달건수 | 목표(30건) | 초과분 | min(초과분, 20) | 지급액 |
|-----------|:------------:|:----------:|:------:|:---------------:|:------:|
| 예시: rider_A | 42건 | ✅ 달성 | 12건 | 12건 | 6,000원 |
| 예시: rider_B | 28건 | ❌ 미달 | — | — | 0원 |
| 예시: rider_C | 55건 | ✅ 달성 | 25건 | 20건 | 10,000원 |
| **합계** | | | | | **totalPromoCost** |

> `totalPromoCost` = 협력사 전체 라이더 프로모션 합계 → `vr_promo_supp_settlement.TOTAL_PROMO_COST` 와 비교

## STEP 3 — 라이더 주정산 엑셀 업로드

1. 20260617~20260623 기간 라이더 주정산 엑셀 업로드
2. 업로드 완료 후 아래 쿼리로 결과 확인

### 검증 쿼리 ①: 라이더별 PROMOTION 컬럼

```sql
SELECT
    RIDER_ID,
    DELIVERY_CNT,
    DELIVERY_VAL,
    ADD_VAL,
    PROMOTION,                -- ← 프로모션 지급액 (계산값과 일치 여부 확인)
    TAXABLE_DELIVERY_VAL,     -- ← 과세배달료 (PROMOTION 포함됨)
    TOT_ACCT_VAL
FROM vr_rider_acct_weekly
WHERE SUPPLY_ID = 'DP2503071840'
  AND ACCT_ST_DATE = '20260617'
ORDER BY PROMOTION DESC;
```

**확인 포인트:**
- `PROMOTION` = STEP 2 기대값과 일치하는지
- `TAXABLE_DELIVERY_VAL` = `(DELIVERY_VAL + ADD_VAL + PROMOTION - TOTAL_MANAGE_FEE)` 를 10원 단위 내림한 값인지

### 검증 쿼리 ②: promo_supp_settlement PENDING 상태

```sql
SELECT
    SUPP_ID,
    ACCT_ST_DATE,
    ACCT_ED_DATE,
    TOTAL_PROMO_COST,   -- ← STEP 2 합계와 일치 여부
    HOLD_STATUS         -- ← 'PENDING' 이어야 함
FROM vr_promo_supp_settlement
WHERE SUPP_ID = 'DP2503071840';
```

## STEP 4 — 협력사 주정산 엑셀 업로드

1. 20260617~20260623 기간 협력사 주정산 엑셀 업로드
2. 업로드 완료 후 아래 쿼리로 상태 전이 확인

### 협력사 주정산비용(ACCT_VAL) 확인

```sql
SELECT
    SUPPLY_ID,
    ACCT_ST_DATE,
    ACCT_ED_DATE,
    ACCT_VAL    -- ← 협력사가 본사로부터 받는 주정산비용
FROM vr_supp_acct_weekly
WHERE SUPPLY_ID = 'DP2503071840'
  AND ACCT_ST_DATE = '20260617';
```

### 검증 쿼리 ③: HELD vs PENDING_DEPOSIT 상태 전이

```sql
SELECT
    SUPP_ID,
    TOTAL_PROMO_COST,
    PARTNER_ACCT_VAL,
    HOLD_STATUS,    -- 'HELD' 또는 'PENDING_DEPOSIT'
    NOTICE_ID       -- PENDING_DEPOSIT 시 공지 ID 존재
FROM vr_promo_supp_settlement
WHERE SUPP_ID = 'DP2503071840';
```

**판단 기준:**
- `PARTNER_ACCT_VAL >= TOTAL_PROMO_COST` → `HOLD_STATUS = HELD`
- `PARTNER_ACCT_VAL < TOTAL_PROMO_COST` → `HOLD_STATUS = PENDING_DEPOSIT`, `NOTICE_ID` 존재

## STEP 5 — 정산 Apply (적용)

1. 업로드 화면에서 협력사 주정산 업로드 건 **적용** 버튼 클릭
2. 아래 쿼리로 결과 확인

### 검증 쿼리 ④: vr_account_mst PD 레코드 (HELD인 경우)

```sql
SELECT
    ACCT_TYPE,      -- 'PD' 이어야 함
    ACCT_DATE,
    SUPPLY_ID,
    TRAN_AMT,       -- ← -TOTAL_PROMO_COST (음수)
    BAL_AMT         -- ← 이전 잔액 - TOTAL_PROMO_COST
FROM vr_account_mst
WHERE SUPPLY_ID = 'DP2503071840'
  AND ACCT_TYPE = 'PD'
ORDER BY SEQ_NO DESC
LIMIT 1;
```

### 검증 쿼리 ⑤: promo_supp_settlement PROCESSED

```sql
SELECT HOLD_STATUS FROM vr_promo_supp_settlement
WHERE SUPP_ID = 'DP2503071840';
-- 'PROCESSED' 이어야 함
```
