## 1. 슬롯(Slot)이란?

슬롯은 **시간대별 배달 물량 목표치 달성**을 나타내는 단위입니다.

하루를 4개의 시간대로 구분하고, 각 시간대에서 목표 건수 이상 배달을 완료하면 해당 슬롯을 달성한 것으로 인정합니다.

### 4가지 슬롯 타입

| 코드  | 명칭         | 시간대      |
|-------|-------------|------------|
| SLOT1 | 아침점심피크 (MorTime)  | 오전 |
| SLOT2 | 오후논피크 (AfterNon)   | 오후 |
| SLOT3 | 저녁피크 (Night)        | 저녁 |
| SLOT4 | 심야논피크 (NightNon)   | 심야 |

---

## 2. SLOT_SET_CNT (협력사 계약 세트 수)

### 정의

`vr_supplier_info.SLOT_SET_CNT` — 협력사가 계약한 슬롯 세트 개수

### 역할

협력사마다 **목표치를 차등 적용**하기 위한 승수(배수) 값입니다.

```
협력사 시간대별 목표치 = 기본 목표치(공통코드) × SLOT_SET_CNT
```

### 예시

- 기본 SLOT1 목표 (일요일): 20건
- 협력사 A의 SLOT_SET_CNT = 2
- → 협력사 A의 일요일 SLOT1 목표 = 20 × 2 = **40건**
- → 40건 이상 완료 시 SLOT1 달성 ✅

---

## 3. 슬롯 달성률 계산

### 일별 모드

```
슬롯 달성률 = 달성한 시간대 수 / 4 × 100

예) 아침·저녁만 달성 → 2 / 4 = 50%
```

### 주별 모드 (활성 협력사 기준)

활성 협력사 = 해당 기간 내 배달 완료 1건 이상인 협력사만 포함

```
슬롯 달성률 = 전체 달성한 슬롯 수 / (활성협력사 수 × 기간일수 × 4) × 100

예) 활성 협력사 5개, 7일 기간, 달성 슬롯 60개
  → 60 / (5 × 7 × 4) = 60 / 140 = 42.8%
```

### 핵심 상수 (`DashboardQueryDslRepository.java`)

```java
private static final int SLOT_COUNT = 4;           // 시간대 수
private static final long DEFAULT_SLOT_SET_CNT = 1L; // 슬롯 세트 수 기본값
```

---

## 4. 슬롯 목표치 저장 위치

슬롯별 기본 목표치는 **공통코드(CommonCode)** 테이블에 저장됩니다.

- 그룹코드: `SLOTINFO`
- 코드: `SLOT1`, `SLOT2`, `SLOT3`, `SLOT4`
- 각 코드 값에는 요일별(1~7, 월~일) 목표 건수가 저장됩니다.

### 5.4 슬롯 수정 흐름

#### 개별 수정

```
PUT /api/v1/supplier/basic-info/{suppId}
  → SupplierController.updateSupplierBasicInfo()
  → SupplierRepositoryImpl.updateBasicInfo()
  → SupplierRepositoryImpl.updateSlotSetCount()
  → vr_supplier_info UPDATE
```

#### 엑셀 일괄 수정

```
POST /api/v1/supplier/grade/upload (엑셀 파일)
  → SupplierController.uploadGradeExcel()
  → SupplierUseCase.uploadGradeExcel()
  → SupplierUseCase.parseSlotSetSheet()   // 9번째 컬럼에서 slotSetCnt 추출
  → SupplierUseCase.updateSlotSets()      // 협력사별 일괄 update
  → SupplierRepositoryImpl.updateSlotSetCount() × N
```

### 5.5 슬롯 조회·계산 흐름 (대시보드)

```
GET /api/v1/dashboard/weekly
  → DashboardQueryDslRepository.getWeeklyPerformance()
  
  1. getSupplierSlotSetCount(suppId)
     → vr_supplier_info에서 SLOT_SET_CNT 조회
     → 없으면 기본값 1 사용
  
  2. loadSlotTargets()
     → 공통코드(SLOTINFO)에서 SLOT1~4별 요일별 목표 건수 로드
  
  3. getSlotTarget(slotTargets, slotCode, dayOfWeek, slotSetCnt)
     → 기본목표 × slotSetCnt = 협력사 목표
  
  4. buildDailyPerformances()
     → 실제 건수 >= 목표 건수 → 슬롯 달성(1), 미달(0)
     → 일별 달성률 = 달성 슬롯 / 4 × 100
  
  5. recalculateSlotAchievementForActiveSuppliers()
     → 완료 1건 이상인 협력사만 포함
     → 기간 전체 달성률 계산
```

### 5.6 핵심 메서드 (`DashboardQueryDslRepository`)

## 7. 슬롯 데이터 흐름 요약

```
[공통코드 DB]                    [vr_supplier_info]
SLOTINFO > SLOT1~4              SLOT_SET_CNT
요일별 기본 목표건수              협력사별 세트 수
       ↓                               ↓
       └──────────── × ───────────────┘
                     ↓
            협력사 시간대별 목표건수
                     ↓
            실제 배달 완료건수와 비교
                     ↓
            시간대별 슬롯 달성 여부 (0/1)
                     ↓
            슬롯 달성률 (%) → 대시보드 표시
```

## 8. 배치 서버

`vroong-3pl-platform-batch` 에서 슬롯 관련 배치 Job은 **없음**.  
슬롯 계산은 대시보드 API 호출 시 실시간으로 수행됩니다.
